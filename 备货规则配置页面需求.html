#备货规则配置页面 — PRD

#1. 背景与目标

备货规则配置页面用于配置补货建议量的计算参数，使系统能够基于从计算日开始的每日预测销量、采购时效、物流时效、安全库存天数和库存状态，输出可解释的发货建议量与下单建议量。

本页面的核心目标是：

  • 建立一套清晰、可配置、可追溯的备货规则。
  • 将销量预测模块输出的从计算日开始的每日预测销量曲线用于备货计算。
  • 支持分别计算「发货建议量」与「下单建议量」。
  • 在建议量异常时给出明确提示，帮助用户识别断货风险或库存积压风险。

#2. 页面定位

页面名称：

备货规则配置

页面用途：

配置补货建议量计算所需的规则参数，并提供覆盖天数拆解、计算公式说明和异常提示说明。

适用对象：

  • 产品运营
  • 供应链计划人员
  • 采购计划人员
  • 补货策略配置人员

#3. 功能范围

#3.1 本期范围

本期页面支持配置：

  • 规则基础信息。
  • 规则适用范围。
  • 采购时效参数。
  • FBA 默认安全天数。
  • 安全库存天数调整规则。
  • 直发物流时效。
  • 建议量扣除必断货量开关。
  • 发货建议量与下单建议量计算公式说明。

#4. 依赖关系

备货规则依赖销量预测模块输出的从计算日开始的每日预测销量：

\[\hat{a}_{D}, \hat{a}_{D+1}, \ldots, \hat{a}_{D+n-1}\]

其中：

符号 含义
\(D\) 计算日 / 决策日
\(\hat{a}_{D+i}\) 从计算日起第 \(i+1\) 个自然日的预测日销量，\(i\) 从 0 开始
\(N\) 当前备货场景的覆盖天数

预测销量求和：

\[F(N) = \sum_{i=0}^{N-1} \hat{a}_{D+i}\]

积分区间固定为：

D ~ D+N-1

示例：

覆盖天数 = 70 天
预测销量求和 = Σ forecastSales(D ... D+69)

#5. 页面结构

页面建议按以下模块组织：

  1. 规则基础信息
  2. 适用范围
  3. 采购时效
  4. FBA 安全库存
  5. 安全库存天数调整
  6. 直发物流时效
  7. 建议量计算设置

#8. 采购时效

#8.1 字段

字段 类型 默认值 必填 说明
采购频率 数字 15 天 是 下一次采购机会间隔
采购交期 数字 30 天 是 从下单到供应端交付的周期
提货准备 数字 7 天 是 到货后可发货前的准备时间
质检 数字 2 天 是 到货后质检处理时间

#8.2 计算口径

为避免将「本次下单多久能形成可发货库存」与「下一次采购机会间隔」混淆，采购相关时间拆分为两个口径：

采购交付提前期 = 采购交期 + 提货准备 + 质检
<!-- 人工注释：目前暂时没有直接使用"采购交付提前期"的公式，基本都是"采购周期覆盖天数"参与计算 -->

采购周期覆盖天数 = 采购交付提前期 + 采购频率

其中：

  • 采购交付提前期表示本次下单后，货物最早可进入 CN 仓 / 本地仓可发状态所需的时间。
  • 采购频率表示下一次采购机会间隔，不代表本次下单的交付时间，但需要纳入下单建议量的覆盖周期。
  • 页面需实时展示采购交付提前期和采购周期覆盖天数两个汇总值。

示例：

采购交付提前期 = 30 + 7 + 2 = 39 天
采购周期覆盖天数 = 39 + 15 = 54 天

后续公式中，如无特别说明，「采购周期覆盖天数」用于下单覆盖天数，「采购交付提前期」用于判断本次下单动作的最早生效时间。

#8.3 校验

  • 采购交期、提货准备、质检必须为大于等于 0 的整数。
  • 采购频率建议为大于 0 的整数；若配置为 0，需明确解释为「可随时下单，无固定采购间隔」。
  • 建议配置合理上限，例如不超过 365 天。
  • 若输入为空，保存时需提示用户补全。

#9. FBA 安全库存

#9.1 字段

字段 类型 默认值 必填 说明
FBA 默认安全天数 数字 28 天 是 未命中安全库存调整条件时使用

#9.2 业务含义

FBA 默认安全天数表示 FBA 端需要保持的最低库存缓冲天数，用于抵御销量波动、物流延误等风险。

#9.3 校验

  • 必须为大于等于 0 的整数。
  • 建议配置合理上限，例如不超过 180 天。

#10. 安全库存天数调整

#10.1 规则说明

当父体 / SKU 命中多个安全库存条件时，最终安全天数取所有命中条件中的最高值：

最终安全天数 = max(默认安全天数, 所有命中条件安全天数)

不按优先级覆盖，不叠加。

#10.2 配置维度

安全库存天数调整按「分类」独立配置。每个分类可单独启用 / 停用，并可在分类下维护多条条件。系统根据父体 / SKU 命中的条件取最高安全天数。

分类 默认条件 默认安全天数 备注
销售层级 A - 爆款 45 天 补货激进
销售层级 B - 旺款 28 天 补货稳健
销售层级 C - 利润款 20 天 补货保守
生命周期 新品（0-3个月） 45 天 新品爬坡期，降低早期断货风险
生命周期 新品（3-9个月） 32 天 新品爬坡期，降低早期断货风险
生命周期 老品（9个月以上） 20 天 新品爬坡期，降低早期断货风险
颜色标签 主色 45 天 核心颜色，优先保障库存
颜色标签 次色 28 天 常规颜色，按稳健安全天数覆盖
流量 OR 自然流量 ASIN 45 天 自然流量入口，优先保障库存
流量 AD 广告流量 ASIN 28 天 广告承接款，按稳健安全天数覆盖

分类配置要求：

  • 销售层级、生命周期、颜色标签、流量为彼此独立的配置分类。
  • 每个分类均支持独立启用 / 停用。
  • 每个分类下均支持新增、编辑、删除条件行。
  • 条件名称为下拉选项，不支持自由文本输入。
  • 每条条件至少包含条件名称、安全天数、启用状态。
  • 同一分类下条件名称不允许重复。
  • 当某分类下所有可选条件均已添加时，新增条件入口置灰或隐藏。
  • 停用的分类或条件不参与最终安全天数计算。

默认下拉选项：

分类 条件名称下拉选项
销售层级 A - 爆款、B - 旺款、C - 利润款
生命周期 新品（0-3个月）、新品（3-9个月）、老品（9个月以上）
颜色标签 主色、次色、边缘色、淘汰色
流量 OR 自然流量 ASIN、AD 广告流量 ASIN

本期默认仅提供上表中的条件名称选项。若后续需要新增其他选项，需先扩展对应分类的条件字典，再允许用户在页面中选择。

#10.3 示例

某 SKU 命中：

命中条件 安全天数
B - 旺款 28 天
新品（0-3 月） 45 天
AD 广告流量 ASIN 28 天

最终安全天数：

max(28, 45, 28) = 45 天

#10.4 交互

  • 各安全天数调整维度可独立启用 / 停用。
  • 用户在分类下新增条件时，先从该分类的条件名称下拉选项中选择条件，再配置安全天数和启用状态。
  • 条件名称下拉选项按分类过滤，不允许跨分类选择。
  • 分类停用时，该分类下全部条件不参与最终安全天数计算。
  • 条件停用时，仅该条件不参与最终安全天数计算。
  • 页面需在每个分类内展示条件列表、启用状态、安全天数和操作入口。

#11. 直发物流时效

#11.1 业务链路

本期仅支持直发链路：

CN 仓 / 本地仓 → FBA 仓

#11.2 字段

字段 类型 必填 说明
大站点 选择 / 文本 是 例如北美、欧洲
本地仓发货频率 数字 是 下一次发货机会间隔
运输方式 选择 / 文本 是 例如快船、慢船、空派
CN 仓至 FBA 时效 数字 是 从发货到 FBA 可用的物流周期

#11.3 默认配置示例

大站点 发货频率 运输方式 物流时效
北美 7 天 快船 35 天
北美 7 天 慢船 55 天
欧洲 7 天 快船 40 天
欧洲 7 天 慢船 60 天

#11.4 交互

  • 支持按站点维护多条运输方式。
  • 同一站点至少保留一条运输方式。
  • 发货频率按站点配置。
  • 物流时效按运输方式配置。

#11.5 校验

  • 发货频率必须为大于等于 0 的整数。
  • 物流时效必须为大于等于 0 的整数。
  • 同一站点下运输方式名称不建议重复。

#12. 建议量计算设置

#12.1 字段

字段 类型 默认值 说明
建议量扣除必断货量 开关 开启 是否从建议量中扣除本次动作无法挽回的预计断货损失

#12.2 业务含义

当库存投影显示部分销量在本次发货 / 下单动作生效前已经无法满足时，这部分销量不应继续计入本次建议量，否则会把已经无法挽回的销量误认为仍可由本次动作覆盖。

必断货扣除量按动作类型分别计算：

  • 发货建议量对应发货动作，扣除发货动作最早可用日前无法挽回的断货损失。
  • 下单建议量对应采购下单动作，扣除下单动作最早可用日前无法挽回的断货损失。
  • 若关闭「建议量扣除必断货量」，对应扣除量按 0 处理，但页面仍建议展示库存投影风险。

必断货扣除量：

必断货扣除量 = Σ 本次动作无法挽回日期内的当日断货损失

#13. 发货建议量

#13.1 业务含义

发货建议量用于回答：

当前应从 CN 仓 / 本地仓发多少货到 FBA？

#13.2 覆盖天数

发货动作最早可用天数：

发货最早可用天数 = CN仓至FBA时效

发货最早可用天数表示本次从 CN 仓 / 本地仓发出的货，最早可以成为 FBA 可售库存的时间。

发货覆盖天数：

发货覆盖天数 =
CN仓至FBA时效 + 安全天数 + CN仓发货频率

说明：

  • CN 仓至 FBA 时效表示运输与入仓可用周期。
  • 安全天数表示 FBA 端风险缓冲。
  • CN 仓发货频率表示下一次补货机会间隔，不是运输时间，但需要纳入覆盖周期。
  • 发货最早可用天数用于判断哪些断货损失已经无法由本次发货补足。
  • 发货覆盖天数用于计算本次发货需要覆盖的总需求周期。

#13.3 计算公式

建议发货量 =
ceil(
  Σ forecastSales(D ... D+发货覆盖天数-1)
  − FBA库存
  − FBA在途
  − 发货必断货扣除量
)

最终展示值：

max(0, 建议发货量)

#13.4 库存扣减口径

发货建议量扣减：

  • FBA 库存。
  • FBA 在途。
  • 发货必断货扣除量。

FBA 在途采用全部扣减口径，不按预计到达日过滤。因此页面必须结合库存投影识别「远期库存已扣减但短期仍可能断货」的风险，并在计算结果中展示相关提示。

#14. 下单建议量

#14.1 业务含义

下单建议量用于回答：

当前应向供应端 / 采购端下单多少？

#14.2 覆盖天数

下单动作最早可用天数：

下单最早可用天数 =
采购交付提前期 + CN仓至FBA时效

下单最早可用天数表示本次向供应端 / 采购端下单后，货物最早可以成为 FBA 可售库存的时间。

下单覆盖天数：

下单覆盖天数 =
采购周期覆盖天数 + CN仓至FBA时效 + 安全天数 + CN仓发货频率

其中：

采购交付提前期 = 采购交期 + 提货准备 + 质检
采购周期覆盖天数 = 采购交付提前期 + 采购频率

说明：

  • 采购交付提前期用于判断本次下单动作的最早可用时间。
  • 采购频率表示下一次采购机会间隔，只纳入覆盖周期，不纳入本次下单动作的最早可用时间。
  • CN 仓发货频率表示下一次发货机会间隔，只纳入覆盖周期，不纳入本次下单动作的最早可用时间。

#14.3 计算公式

建议下单量 =
ceil(
  Σ forecastSales(D ... D+下单覆盖天数-1)
  − FBA库存
  − FBA在途
  − CN仓可用
  − 已下单未入库
  − 下单必断货扣除量
)

最终展示值：

max(0, 建议下单量)

#14.4 库存扣减口径

下单建议量扣减：

  • FBA 库存。
  • FBA 在途。
  • CN 仓可用。
  • 已下单未入库。
  • 下单必断货扣除量。

FBA 在途与已下单未入库采用全部扣减口径，不按预计到达日过滤。因此页面必须结合库存投影识别「远期库存已扣减但短期仍可能断货」的风险，并在计算结果中展示相关提示。

#15. 必断货扣除量

#15.1 推导方式

必断货日期与必断货扣除量由库存投影推断，并按发货动作、下单动作分别计算。

库存投影示意：

当日日初可售库存 =
max(0, 前一日期末可售库存) + 当日预计可用入库

当日可满足销量 =
min(当日日初可售库存, 当日预测销量)

当日断货损失 =
当日预测销量 − 当日可满足销量

当日期末可售库存 =
max(0, 当日日初可售库存 − 当日预测销量)

等价地，当日断货损失也可写为：

当日断货损失 =
min(当日预测销量, max(0, 当日预测销量 − 当日日初可售库存))

这样可以避免将当天已经能够满足的销量也计入断货损失。

当某日当日断货损失大于 0，且该损失无法由本次发货 / 下单动作在当日销售前补足，则该日视为对应动作的预计必断货日。

默认口径：

  • 当日预计可用入库在当日销售前可用。
  • 若动作最早可用天数为 L，则 D+L 当日可参与满足销量，D ~ D+L-1 内的断货损失视为本次动作无法挽回。
  • 若 L = 0，则本次动作在计算日 D 当日即可参与满足销量，不存在动作生效前的不可挽回区间。
  • 若数据源口径为日末入库，则需将不可挽回区间调整为 D ~ D+L。

#15.2 计算公式

发货必断货扣除量：

发货必断货扣除量 =
Σ 发货最早可用日前无法挽回的当日断货损失

下单必断货扣除量：

下单必断货扣除量 =
Σ 下单最早可用日前无法挽回的当日断货损失

若「建议量扣除必断货量」开关关闭，则：

发货必断货扣除量 = 0
下单必断货扣除量 = 0

但页面仍需保留预计断货风险提示，避免用户误以为库存投影无风险。

#15.3 展示要求

若必断货扣除量大于 0，页面需展示：

  • 预计断货日期范围。
  • 断货日期内每日预测销量。
  • 断货日期内每日可满足销量。
  • 断货日期内每日断货损失。
  • 必断货扣除量合计。
  • 对发货建议量 / 下单建议量的影响。

提示文案：

存在预计必断货区间，已根据库存投影扣除本次动作无法挽回的断货损失。

#16. 异常提示

#16.1 建议量小于等于 0

若原始建议量小于或等于 0：

最终建议量展示为 0

提示文案：

当前库存及在途库存已覆盖目标周期，建议量为 0，请关注库存积压或远期到货风险。

#16.2 必断货扣除量较大

若必断货扣除量占覆盖周期预测销量比例达到配置阈值，页面需高亮提示。

推荐判断：

必断货扣除量 / 覆盖周期预测销量 ≥ 阈值

推荐默认阈值：

等级 判断条件 展示建议
提醒 ≥ 10% 普通高亮提示
严重 ≥ 25% 强高亮提示，并建议用户查看库存投影

提示文案：

预计断货影响较大，本次建议量已扣除无法挽回的断货损失，请关注到货节奏。

#16.3 远期库存风险

由于 FBA 在途、已下单未入库采用全部扣减口径，不按预计到达日过滤，若建议量为 0 但库存投影仍存在短期断货，页面需提示：

部分库存预计到达时间较晚，虽然已在建议量中扣减，但短期仍可能出现断货，请关注库存投影。

触发条件建议：

最终建议量 = 0
且
库存投影在覆盖周期内存在当日断货损失 > 0

该提示不改变建议量，仅用于说明全量扣减远期库存带来的短期风险。

#19. 验收标准

#19.1 配置验收

  • 用户可以创建、编辑、启用、停用备货规则。
  • 用户可以配置采购频率、采购交期、提货准备、质检。
  • 页面能实时展示采购交付提前期和采购周期覆盖天数。
  • 用户可以配置默认安全天数。
  • 用户可以配置并启停各安全库存天数调整维度。
  • 用户可以按站点维护直发物流时效。
  • 用户可以配置是否扣除必断货量。

#19.2 计算验收

  • 采购交付提前期按以下公式计算：
采购交期 + 提货准备 + 质检
  • 采购周期覆盖天数按以下公式计算：
采购交付提前期 + 采购频率
  • 发货最早可用天数按以下公式计算：
CN仓至FBA时效
  • 下单最早可用天数按以下公式计算：
采购交付提前期 + CN仓至FBA时效
  • 发货覆盖天数按以下公式计算：
CN仓至FBA时效 + 安全天数 + CN仓发货频率
  • 下单覆盖天数按以下公式计算：
采购周期覆盖天数 + CN仓至FBA时效 + 安全天数 + CN仓发货频率
  • 预测销量求和区间为：
D ~ D+N-1
  • 建议量向上取整数。
  • 最终建议量不得小于 0。
  • 安全天数取所有命中条件中的最高值。
  • 当日断货损失基于当日预测销量、当日日初可售库存、当日可满足销量推导。
  • 发货必断货扣除量按发货最早可用日前无法挽回的当日断货损失求和。
  • 下单必断货扣除量按下单最早可用日前无法挽回的当日断货损失求和。
  • 关闭「建议量扣除必断货量」时，发货 / 下单必断货扣除量按 0 参与建议量计算。

#19.3 提示验收

  • 必断货扣除量大于 0 时，页面必须展示预计断货日期、每日断货损失和扣除量合计。
  • 必断货扣除量占覆盖周期预测销量比例达到提醒 / 严重阈值时，页面必须分级高亮提示。
  • 原始建议量小于等于 0 时，页面展示 0 并提示库存覆盖风险。
  • 全量扣减远期库存但短期仍可能断货时，页面必须提示远期库存风险。
