# Project Defaults

## Validation Policy

- For this project, code and HTML changes should not trigger automatic validation by default.
- Do not run browser checks, screenshots, visual inspection, syntax checks, test commands, lint commands, or static verification commands unless the user explicitly asks for them.
- Generating requested artifacts, such as converting PRD Markdown into HTML, is allowed because it is part of the requested output rather than validation.
- In the final response, state that validation was skipped by project default when relevant.
