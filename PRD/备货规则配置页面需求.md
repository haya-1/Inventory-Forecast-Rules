# 备货规则配置页面 — PRD

## 1. 背景与目标

备货规则配置页面用于配置补货建议量的计算参数，使系统能够基于从计算日开始的每日预测销量、采购时效、物流时效、安全库存天数和库存状态，输出可解释的发货建议量与下单建议量。

本页面的核心目标是：

- 建立一套清晰、可配置、可追溯的备货规则。
- 将销量预测模块输出的从计算日开始的每日预测销量曲线用于备货计算。
- 支持分别计算「发货建议量」与「下单建议量」。
- 在建议量异常时给出明确提示，帮助用户识别断货风险或库存积压风险。

---

## 2. 页面定位

页面名称：

```text
备货规则配置
```

页面用途：

```text
配置补货建议量计算所需的规则参数，并提供覆盖天数拆解、计算公式说明和异常提示说明。
```

适用对象：

- 产品运营
- 供应链计划人员
- 采购计划人员
- 补货策略配置人员

## 4. 依赖关系

备货规则依赖销量预测模块输出的从计算日开始的每日预测销量：

\[
\hat{a}_{D}, \hat{a}_{D+1}, \ldots, \hat{a}_{D+n-1}
\]

其中：

| 符号 | 含义 |
|------|------|
| \(D\) | 计算日 / 决策日 |
| \(\hat{a}_{D+i}\) | 从计算日起第 \(i+1\) 个自然日的预测日销量，\(i\) 从 0 开始 |
| \(N\) | 当前备货场景的覆盖天数 |

预测销量求和：

\[
F(N) = \sum_{i=0}^{N-1} \hat{a}_{D+i}
\]

积分区间固定为：

```text
D ~ D+N-1
```

示例：

```text
覆盖天数 = 70 天
预测销量求和 = Σ forecastSales(D ... D+69)
```

## 6. 规则基础信息

### 6.1 字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 规则名称 | 文本 | 是 | 用户自定义备货规则名称 |
| 规则编码 | 文本 / 系统生成 | 是 | 用于唯一识别规则 |
| 规则状态 | 开关 | 是 | 启用 / 停用 |
| 备注 | 文本 | 否 | 规则说明 |

### 6.2 交互

- 新增规则时，规则状态默认启用。
- 停用规则不参与补货建议量计算。
- 规则编码可由系统生成，用户不可编辑。

### 6.3 目标库存天数实时预览

基础信息下方展示「目标库存天数（按当前配置实时预览）」卡片，用于让用户在配置阶段理解当前规则会形成多长的目标库存周期。

预览分为两个动作：

```text
采购目标库存天数 =
生产周期 + 质检 + 提货准备 + 最长物流时效 + 安全天数 + 采购频率

发货目标库存天数 =
最长物流时效 + 安全天数 + 发货频率
```

说明：

- 生产周期默认展示为 30 天，实际计算时取自 PLM 或基础参数来源。
- 质检、提货准备、采购频率、发货频率由「基础参数配置」维护，备货规则页只读展示。
- 最长物流时效取当前规则物流时效页签中已配置物流方式的最大时效。
- 安全天数取当前默认安全天数；实际子体计算时仍以子体最终安全天数为准。
- 当用户调整默认安全天数、物流方式、物流优先级或基础参数回填值时，预览需实时刷新。

示例：

```text
生产周期 30 + 质检 2 + 提货准备 3 + 最长物流 57 + 安全天数 28 + 采购频率 15 = 135 天
最长物流 57 + 安全天数 28 + 发货频率 7 = 92 天
```

## 8. 采购时效

### 8.1 字段

| 字段 | 类型 | 默认值 | 必填 | 说明 |
|------|------|--------|------|------|
| 采购频率 | 只读数字 | 15 天 | 是 | 下一次采购机会间隔，由基础参数配置维护 |
| 采购交期 / 生产周期 | 只读数字 | 30 天 | 是 | 从下单到供应端交付的周期，实际可取自 PLM 或基础参数 |
| 提货准备 | 只读数字 | 3 天 | 是 | 到货后可发货前的准备时间，由基础参数配置维护 |
| 质检 | 只读数字 | 2 天 | 是 | 到货后质检处理时间，由基础参数配置维护 |

### 8.2 计算口径

为避免将「本次下单多久能形成可发货库存」与「下一次采购机会间隔」混淆，采购相关时间拆分为两个口径：

```text
采购交付提前期 = 采购交期 + 提货准备 + 质检
<!-- 人工注释：目前暂时没有直接使用"采购交付提前期"的公式，基本都是"采购周期覆盖天数"参与计算 -->
```

```text
采购周期覆盖天数 = 采购交付提前期 + 采购频率
```

其中：

- 采购交付提前期表示本次下单后，货物最早可进入 CN 仓 / 本地仓可发状态所需的时间。
- 采购频率表示下一次采购机会间隔，不代表本次下单的交付时间，但需要纳入下单建议量的覆盖周期。
- 页面需实时展示采购交付提前期和采购周期覆盖天数两个汇总值；若采用目标库存天数预览卡片，则以采购目标库存天数公式展示给用户。

示例：

```text
采购交付提前期 = 30 + 3 + 2 = 35 天
采购周期覆盖天数 = 35 + 15 = 50 天
```

后续公式中，如无特别说明，「采购周期覆盖天数」用于下单覆盖天数，「采购交付提前期」用于判断本次下单动作的最早生效时间。

### 8.3 校验

- 采购交期、提货准备、质检由 PLM 或基础参数配置维护，本页只读展示。
- 基础参数回填到本页时，采购交期、提货准备、质检必须为大于等于 0 的整数。
- 采购频率由基础参数配置维护；若配置为 0，需明确解释为「可随时下单，无固定采购间隔」。
- 建议配置合理上限，例如不超过 365 天。
- 若输入为空，保存时需提示用户补全。

## 9. FBA 安全库存

### 9.1 字段

| 字段 | 类型 | 默认值 | 必填 | 说明 |
|------|------|--------|------|------|
| FBA 默认安全天数 | 数字 | 28 天 | 是 | 未命中安全库存调整条件时使用 |

### 9.2 业务含义

FBA 默认安全天数表示 FBA 端需要保持的最低库存缓冲天数，用于抵御销量波动、物流延误等风险。

### 9.3 校验

- 必须为大于等于 0 的整数。
- 建议配置合理上限，例如不超过 180 天。

## 10. 安全库存天数调整

### 10.1 规则说明

当父体 / SKU 命中多个安全库存条件时，最终安全天数取所有命中条件中的最高值：

```text
最终安全天数 = max(默认安全天数, 所有命中条件安全天数)
```

不按优先级覆盖，不叠加。

### 10.2 配置维度

安全库存天数调整按「分类」独立配置。每个分类可单独启用 / 停用，并可在分类下维护多条条件。系统根据父体 / SKU 命中的条件取最高安全天数。

| 分类 | 默认条件 | 默认安全天数 | 备注 |
|------|----------|--------------|------|
| 销售层级 | A - 爆款 | 45 天 | 补货激进 |
| 销售层级 | B - 旺款 | 28 天 | 补货稳健 |
| 销售层级 | C - 利润款 | 20 天 | 补货保守 |
| 生命周期 | 新品（0-3个月） | 45 天 | 新品爬坡期，降低早期断货风险 |
| 生命周期 | 新品（3-9个月） | 32 天 | 新品爬坡期，降低早期断货风险 |
| 生命周期 | 老品（9个月以上） | 20 天 | 新品爬坡期，降低早期断货风险 |
| 颜色标签 | 主色 | 45 天 | 核心颜色，优先保障库存 |
| 颜色标签 | 次色 | 28 天 | 常规颜色，按稳健安全天数覆盖 |
| 流量 | OR 自然流量 ASIN | 45 天 | 自然流量入口，优先保障库存 |
| 流量 | AD 广告流量 ASIN | 28 天 | 广告承接款，按稳健安全天数覆盖 |

分类配置要求：

- 销售层级、生命周期、颜色标签、流量为彼此独立的配置分类。
- 每个分类均支持独立启用 / 停用。
- 每个分类下均支持新增、编辑、删除条件行。
- 条件名称为下拉选项，不支持自由文本输入。
- 每条条件至少包含条件名称、安全天数、启用状态。
- 同一分类下条件名称不允许重复。
- 当某分类下所有可选条件均已添加时，新增条件入口置灰或隐藏。
- 停用的分类或条件不参与最终安全天数计算。

默认下拉选项：

| 分类 | 条件名称下拉选项 |
|------|------------------|
| 销售层级 | A - 爆款、B - 旺款、C - 利润款 |
| 生命周期 | 新品（0-3个月）、新品（3-9个月）、老品（9个月以上） |
| 颜色标签 | 主色、次色、边缘色、淘汰色 |
| 流量 | OR 自然流量 ASIN、AD 广告流量 ASIN |

本期默认仅提供上表中的条件名称选项。若后续需要新增其他选项，需先扩展对应分类的条件字典，再允许用户在页面中选择。

### 10.3 示例

某 SKU 命中：

| 命中条件 | 安全天数 |
|----------|----------|
| B - 旺款 | 28 天 |
| 新品（0-3 月） | 45 天 |
| AD 广告流量 ASIN | 28 天 |

最终安全天数：

```text
max(28, 45, 28) = 45 天
```

### 10.4 交互

- 各安全天数调整维度可独立启用 / 停用。
- 用户在分类下新增条件时，先从该分类的条件名称下拉选项中选择条件，再配置安全天数和启用状态。
- 条件名称下拉选项按分类过滤，不允许跨分类选择。
- 分类停用时，该分类下全部条件不参与最终安全天数计算。
- 条件停用时，仅该条件不参与最终安全天数计算。
- 页面需在每个分类内展示条件列表、启用状态、安全天数和操作入口。

### 11.4 交互

- 支持按站点维护多条物流方式。
- 同一站点至少保留一条物流方式。
- 发货频率按站点配置。
- 物流方式为下拉选择，选项来自外部物流参数。
- 物流时效按选中的物流方式自动展示，不允许手动修改。
- 计算优先级用于多物流方式建议量分配顺序，可由用户调整；数字越小优先级越高。

### 11.5 多运输方式计算口径

当同一站点配置多种运输方式时，系统需要按运输方式分别输出建议量。

多运输方式计算不是对每种运输方式都重复计算一整段覆盖周期，而是按物流方式计算优先级依次认领覆盖周期内可被该物流方式覆盖的日期需求：

- 先按计算优先级从高到低排序运输方式；数字越小优先级越高。
- 若优先级相同，按物流时效从短到长作为兜底排序。
- 每种运输方式只能覆盖其到货可用日及之后的需求，不能覆盖到货日前的销量。
- 系统按优先级顺序计算当前运输方式可覆盖区间内尚未被更高优先级运输方式覆盖的日期需求。
- 当覆盖周期内所有可被运输方式覆盖的日期均已被认领后，停止后续低优先级运输方式计算。
- 最短物流方式到货日前的预测销量不归属于任何运输方式建议量，仅用于库存投影和发货窗口提示，不作为扣减项影响建议量。

示例：

```text
运输方式：空运 15 天，普船 45 天
发货覆盖周期结束日：D+90
优先级：空运 = 1，普船 = 2

空运建议量覆盖：D+15 ~ D+90
普船建议量覆盖：0（可覆盖日期已被更高优先级空运覆盖）
```

若优先级调整为普船 = 1，空运 = 2，则：

```text
普船建议量覆盖：D+45 ~ D+90
空运建议量覆盖：D+15 ~ D+44
```

如果只有一种运输方式，则该运输方式覆盖从其到货日至覆盖周期结束的全部需求。

库存扣减口径：

- FBA 库存、FBA 在途、本地仓库存、已下单未入库等库存类扣减项不能在每种运输方式上重复扣减。
- 系统应基于库存投影或统一缺口分摊口径，将可用库存优先用于较早日期需求，再按运输方式分段得到各运输方式建议量。
- 各运输方式建议量求和应等于当前动作维度下的最终建议量，允许因向上取整存在极小差异，差异需落到最后一种运输方式上。

### 11.6 校验

- 发货频率必须为大于等于 0 的整数。
- 物流时效由外部系统提供，页面不做手动填写校验。
- 同一站点下物流方式不允许重复。
- 同一站点下启用且参与计算的物流方式不允许出现重复优先级；保存时若重复，需阻止提交并提示用户调整。
- 若历史数据已存在重复优先级，系统读取时可按物流时效从短到长作为兜底排序，但编辑提交前必须修正为唯一优先级。

## 12. 计算设置

### 12.1 字段

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 年规目标 | 勾选项 | 勾选且不可取消 | 使用年规目标作为建议量计算来源之一 |
| 预测日销量 | 勾选项 | 勾选 | 使用销量预测模块输出的预测日销量作为建议量计算来源之一 |

### 12.2 业务含义

页面允许同时输出年规目标和预测日销量两类计算结果。年规目标为必选来源，预测日销量可按业务需要启用或关闭。

发货建议量和下单建议量均不再扣减必断货扣除量。库存投影推断出的断货损失只用于库存走势解释、发货窗口或后续预警模块，不作为建议量计算扣减项。

## 13. 发货建议量

### 13.1 业务含义

发货建议量用于回答：

```text
当前应从 CN 仓 / 本地仓发多少货到 FBA？
```

### 13.2 覆盖天数

发货动作最早可用天数：

```text
发货最早可用天数 = CN仓至FBA时效
```

发货最早可用天数表示本次从 CN 仓 / 本地仓发出的货，最早可以成为 FBA 可售库存的时间。

发货覆盖天数：

```text
发货覆盖天数 =
CN仓至FBA时效 + 安全天数 + CN仓发货频率
```

当同一站点仅配置一种运输方式时，按该运输方式的物流时效计算发货覆盖天数。

当同一站点配置多种运输方式时，发货覆盖周期以最长物流时效作为周期终点基础：

```text
发货覆盖天数 =
最长CN仓至FBA时效 + 安全天数 + CN仓发货频率
```

说明：

- CN 仓至 FBA 时效表示运输与入仓可用周期。
- 安全天数表示 FBA 端风险缓冲。
- CN 仓发货频率表示下一次补货机会间隔，不是运输时间，但需要纳入覆盖周期。
- 发货最早可用天数用于发货窗口判断与多物流方式可覆盖日期判断，不再用于计算发货必断货扣除量。
- 发货覆盖天数用于计算本次发货需要覆盖的总需求周期。

### 13.3 多运输方式优先级分配

若发货规则命中多种运输方式，按物流方式计算优先级进行分配：

```text
第 k 种运输方式发货最早可用天数 = 该运输方式 CN仓至FBA时效

第 k 种运输方式理论可覆盖区间 =
D + 第 k 种运输方式发货最早可用天数 ~ D + 发货覆盖天数

第 k 种运输方式实际负责区间 =
理论可覆盖区间内尚未被更高优先级运输方式覆盖的日期
```

示例：

```text
空运 15 天，普船 45 天，发货覆盖天数 90 天
优先级：空运 = 1，普船 = 2

空运建议量覆盖：D+15 ~ D+90
普船建议量覆盖：0（可覆盖日期已被更高优先级空运覆盖）
```

若优先级调整为普船 = 1，空运 = 2：

```text
普船建议量覆盖：D+45 ~ D+90
空运建议量覆盖：D+15 ~ D+44
```

发货建议量需要同时输出总量与各运输方式明细：

```text
发货建议总量 = Σ 各运输方式发货建议量
```

### 13.4 计算公式

```text
建议发货量 =
ceil(
  Σ forecastSales(D ... D+发货覆盖天数-1)
  − FBA库存
  − FBA在途
)
```

最终展示值：

```text
max(0, 建议发货量)
```

多运输方式下，每种运输方式建议量基于其按优先级认领的实际负责日期段内的预测销量计算。到货前累计销量只参与库存投影和发货窗口判断，不直接计入该运输方式建议量，也不作为发货必断货扣除量从建议量中扣减。库存扣减项不得按运输方式重复扣减，应通过库存投影或统一缺口分摊后落到对应运输方式。

### 13.5 库存扣减口径

发货建议量扣减：

- FBA 库存。
- FBA 在途。

FBA 在途建议按预计到货时间参与库存投影，仅将覆盖周期内可用的在途库存纳入本次建议量扣减。若仍采用全量扣减口径，由此产生的远期库存风险提示不在备货规则配置页闭环，后续由预警模块承接。

## 14. 下单建议量

### 14.1 业务含义

下单建议量用于回答：

```text
当前应向供应端 / 采购端下单多少？
```

### 14.2 覆盖天数

下单动作最早可用天数：

```text
下单最早可用天数 =
采购交付提前期 + CN仓至FBA时效
```

下单最早可用天数表示本次向供应端 / 采购端下单后，货物最早可以成为 FBA 可售库存的时间。

下单覆盖天数：

```text
下单覆盖天数 =
采购周期覆盖天数 + CN仓至FBA时效 + 安全天数 + CN仓发货频率
```

当同一站点仅配置一种运输方式时，按该运输方式的物流时效计算下单覆盖天数。

当同一站点配置多种运输方式时，下单覆盖周期以最长物流时效作为周期终点基础：

```text
下单覆盖天数 =
采购周期覆盖天数 + 最长CN仓至FBA时效 + 安全天数 + CN仓发货频率
```

其中：

```text
采购交付提前期 = 采购交期 + 提货准备 + 质检
采购周期覆盖天数 = 采购交付提前期 + 采购频率
```

说明：

- 采购交付提前期用于判断本次下单动作的最早可用时间。
- 采购频率表示下一次采购机会间隔，只纳入覆盖周期，不纳入本次下单动作的最早可用时间。
- CN 仓发货频率表示下一次发货机会间隔，只纳入覆盖周期，不纳入本次下单动作的最早可用时间。

### 14.3 多运输方式优先级分配

若下单规则命中多种运输方式，按物流方式计算优先级进行分配：

```text
第 k 种运输方式下单最早可用天数 =
采购交付提前期 + 第 k 种运输方式 CN仓至FBA时效

第 k 种运输方式理论可覆盖区间 =
D + 第 k 种运输方式下单最早可用天数 ~ D + 下单覆盖天数

第 k 种运输方式实际负责区间 =
理论可覆盖区间内尚未被更高优先级运输方式覆盖的日期
```

示例：

```text
采购交付提前期 39 天
空运 15 天，普船 45 天，下单覆盖天数 135 天
优先级：空运 = 1，普船 = 2

空运下单最早可用日：D+54
普船下单最早可用日：D+84

空运建议量覆盖：D+54 ~ D+135
普船建议量覆盖：0（可覆盖日期已被更高优先级空运覆盖）
```

若优先级调整为普船 = 1，空运 = 2：

```text
普船建议量覆盖：D+84 ~ D+135
空运建议量覆盖：D+54 ~ D+83
```

下单建议量需要同时输出总量与各运输方式明细：

```text
下单建议总量 = Σ 各运输方式下单建议量
```

### 14.4 计算公式

```text
建议下单量 =
ceil(
  Σ forecastSales(D ... D+下单覆盖天数-1)
  − FBA库存
  − FBA在途
− 本地仓库存
  − 已下单未入库
)
```

最终展示值：

```text
max(0, 建议下单量)
```

多运输方式下，每种运输方式建议量基于其按优先级认领的实际负责日期段内的预测销量计算。到货前累计销量只参与库存投影和发货窗口提示，不直接计入该运输方式建议量，也不作为扣减项影响建议量。库存扣减项不得按运输方式重复扣减，应通过库存投影或统一缺口分摊后落到对应运输方式。

### 14.5 库存扣减口径

下单建议量扣减：

- FBA 库存。
- FBA 在途。
- 本地仓库存。
- 已下单未入库。

FBA 在途与已下单未入库建议按预计到货时间参与库存投影，仅将覆盖周期内可用的库存纳入本次建议量扣减。若仍采用全量扣减口径，由此产生的远期库存风险提示不在备货规则配置页闭环，后续由预警模块承接。

发货建议和采购建议的库存抵扣表均保留完整库存列，包括平台仓、物流在途、本地仓库存、采购在途、总库存。发货建议中的本地仓库存列需展示为：

```text
本地仓库存(预发货+剩余) = 总量(预发货库存 + 剩余库存)
```

采购建议中的本地仓库存列展示普通本地仓库存数值，不拆分预发货库存和剩余库存。该展示不改变发货建议量与采购建议量各自的库存扣减口径。

## 15. 库存投影与断货风险边界

### 15.1 推导方式

系统可基于库存投影推断预计断货点与断货损失，用于解释库存走势、发货窗口或后续预警模块。

发货建议量和下单建议量均不再扣减必断货扣除量。断货损失只作为风险信息，不作为建议量计算扣减项。

库存投影示意：

```text
当日日初可售库存 =
max(0, 前一日期末可售库存) + 当日预计可用入库

当日可满足销量 =
min(当日日初可售库存, 当日预测销量)

当日断货损失 =
当日预测销量 − 当日可满足销量

当日期末可售库存 =
max(0, 当日日初可售库存 − 当日预测销量)
```

等价地，当日断货损失也可写为：

```text
当日断货损失 =
min(当日预测销量, max(0, 当日预测销量 − 当日日初可售库存))
```

这样可以避免将当天已经能够满足的销量也计入断货损失。

默认口径：

- 当日预计可用入库在当日销售前可用。
- 当某日当日断货损失大于 0，可记录为预计断货日或断货风险日。
- 若数据源口径为日末入库，库存投影展示需按实际数据源口径调整。

### 15.2 计算公式

```text
发货必断货扣除量 = 不计算
下单必断货扣除量 = 不计算
```

发货建议量与下单建议量只扣减各自库存类扣减项。断货损失不重复分摊到各物流方式，也不从总建议量中扣减。

### 15.3 展示要求

若库存投影出现断货风险，页面可展示：

- 预计断货日期范围。
- 断货日期内每日预测销量。
- 断货日期内每日可满足销量。
- 断货日期内每日断货损失。

提示文案：

```text
库存投影存在预计断货风险，请关注库存走势与发货窗口；该风险不改变本次建议量。
```

## 16. 异常提示

### 16.1 建议量小于等于 0

若原始建议量小于或等于 0：

```text
最终建议量展示为 0
```

提示文案：

```text
当前库存及在途库存已覆盖目标周期，建议量为 0，请关注库存积压或远期到货风险。
```

### 16.2 断货风险提示边界

断货风险不再作为建议量扣减项，也不在备货规则配置页内输出“必断货扣除量较大”类提示。

若库存投影显示短期断货，但建议量因库存覆盖、在途到货或其他扣减项计算为 0，该风险由发货窗口、库存投影说明或后续预警模块承接。

### 16.3 远期库存风险边界

备货规则配置页不承接远期库存风险提示闭环。若计算层按预计到货时间过滤库存，仅扣减覆盖周期内可用的在途库存，则覆盖周期外远期库存不会压低本次建议量，可不在本页单独提示远期库存风险。

若后续计算层仍采用全量扣减 FBA 在途、已下单未入库等库存口径，短期断货但建议量为 0 的风险由后续预警模块或库存投影模块承接，不作为备货规则配置页必需提示项。

## 19. 验收标准

### 19.1 配置验收

- 用户可以创建、编辑、启用、停用备货规则。
- 用户可以查看采购频率、采购交期 / 生产周期、提货准备、质检等参数，相关字段由 PLM 或基础参数配置维护，本页只读展示。
- 页面能实时展示目标库存天数预览，并随默认安全天数、物流方式与基础参数回填值变化刷新。
- 用户可以配置默认安全天数。
- 用户可以配置并启停各安全库存天数调整维度。
- 用户可以按站点维护直发物流方式、物流时效和计算优先级。

### 19.2 计算验收

- 采购交付提前期按以下公式计算：

```text
采购交期 + 提货准备 + 质检
```

- 采购周期覆盖天数按以下公式计算：

```text
采购交付提前期 + 采购频率
```

- 发货最早可用天数按以下公式计算：

```text
CN仓至FBA时效
```

- 下单最早可用天数按以下公式计算：

```text
采购交付提前期 + CN仓至FBA时效
```

- 发货覆盖天数按以下公式计算：

```text
CN仓至FBA时效 + 安全天数 + CN仓发货频率
```

- 下单覆盖天数按以下公式计算：

```text
采购周期覆盖天数 + CN仓至FBA时效 + 安全天数 + CN仓发货频率
```

- 预测销量求和区间为：

```text
D ~ D+N-1
```

- 建议量向上取整数。
- 最终建议量不得小于 0。
- 安全天数取所有命中条件中的最高值。
- 当日断货损失基于当日预测销量、当日日初可售库存、当日可满足销量推导。
- 发货建议量与下单建议量均不扣减必断货扣除量。
- 断货损失仅用于库存投影、发货窗口或后续预警模块，不改变建议量。
- 同一站点配置多种运输方式时，发货建议量需要按运输方式计算优先级依次认领可覆盖日期并输出拆分明细。
- 同一站点配置多种运输方式时，下单建议量需要按运输方式计算优先级依次认领可覆盖日期并输出拆分明细。
- 多运输方式下，若高优先级运输方式已覆盖全部可覆盖日期，后续低优先级运输方式建议量为 0。
- 多运输方式下，运输方式到货前累计销量只参与库存投影和发货窗口提示，不直接计入该运输方式建议量，也不作为扣减项影响建议量。
- 多运输方式建议量的库存扣减项不得在每种运输方式上重复扣减。

### 19.3 提示验收

- 原始建议量小于等于 0 时，页面展示 0 并提示库存覆盖风险。
- 库存投影断货风险不作为建议量扣减项，后续可由预警模块或库存投影模块承接。
- 远期库存风险不作为备货规则配置页必需提示项，后续由预警模块或库存投影模块承接。

| 日内入库口径 | 当日预计可用入库默认按销售前可用，是否需要按数据源改为日末可用 |
| 物流绑定关系 | 站点、仓库、运输方式与父体 / SKU 的绑定方式 |
| 箱规 / MOQ | 后续是否需要按箱规、MOQ、整箱倍数调整建议量 |
