# 备货建议页面 V0.5 原型绘制指南

## 1. 版本定位

V0.5 是介于 V0 与 V1 之间的试做增强版本。

该版本不做完整的备货工作台，也不引入复杂的处理状态流转，而是采用：

```text
V0 首页 + V1 决策抽屉
```

核心思路：

- 首页保持 V0 的简单结构，负责快速展示计算结果。
- 抽屉吸收 V1 的关键能力，负责解释风险、建议动作和计算依据。
- 暂不做完整优先级队列、状态流转、批量复核和 AI 自动生成内容。

V0.5 的成功标准：

```text
业务能看懂建议量。
业务能相信计算依据。
业务能识别风险。
业务愿意基于建议生成发货计划或采购计划。
```

---

## 2. 设计目标

V0.5 需要回答以下问题：

```text
这个 SKU 建议发多少？
这个 SKU 建议下多少？
为什么是这个数？
有什么风险？
如果不处理会怎样？
下一步建议做什么？
```

页面不追求完整工作台体验，优先保证计算结果清晰、解释路径完整、开发成本可控。

---

## 3. 信息架构

```text
备货建议页面 V0.5
├─ 筛选区
├─ 总览指标
├─ 备货建议列表
```

### 4.2 首页字段建议

筛选区：

| 字段 | 说明 |
|------|------|
| 决策日期 D | 当前建议量计算日期 |
| 站点 | 北美站、欧洲站等 |
| 父体 / SKU | 支持按父体或 SKU 查询 |
| 运输方式 | 快船、慢船、空派等 |
| 适用规则 | 当前命中的备货规则 |
| 只看有建议量 | 过滤发货建议量或下单建议量大于 0 的记录 |
| 只看异常风险 | 过滤断货、远期库存、数据异常等风险记录 |

总览指标：

| 指标 | 说明 |
|------|------|
| 建议发货总量 | 当前筛选条件下建议发货量合计 |
| 建议下单总量 | 当前筛选条件下建议下单量合计 |
| 断货风险 SKU | 存在预计断货损失的 SKU 数 |
| 远期库存风险 | 建议量为 0 但库存投影仍存在短期断货风险的 SKU 数 |

列表字段：

| 字段 | 说明 |
|------|------|
| SKU | 当前建议对象 |
| 站点 | SKU 对应销售站点 |
| 运输方式 | 当前计算使用的运输方式 |
| 标签 | 销售层级 / 生命周期 / 颜色标签 / 流量等摘要 |
| 安全天数 | 最终安全天数 |
| 发货建议量 | 当前建议从 CN 仓 / 本地仓发往 FBA 的数量 |
| 下单建议量 | 当前建议向供应端 / 采购端下单的数量 |
| 风险 | 断货、远期库存、数据异常、正常 |
| 操作 | 查看明细 |

## 5. 右侧决策抽屉原型绘制指南

抽屉采用 V1 的决策解释能力，但做精简。

抽屉采用「固定顶部 + 三段滚动内容」结构。

固定顶部始终展示当前对象和关键动作，滚动内容按决策优先级分为三段：

```text
固定顶部：SKU 信息 + 风险等级 + 建议动作 + 核心操作
第一段：风险说明 + 不处理后果
第二段：库存投影
第三段：计算明细
```

这样用户打开抽屉时，先看到「现在该做什么」；继续向下滚动时，再看「风险从哪里来」和「数字怎么算出来」。

### 5.1 抽屉线框图


### 5.2 抽屉模块说明

建议动作：

| 字段 | 说明 |
|------|------|
| 风险等级 | 高 / 中 / 低 / 数据异常 |
| 建议动作 | 建议生成发货计划、建议生成采购计划、建议复核在途、暂不处理 |
| 建议发货量 | 发货最终建议量 |
| 建议下单量 | 下单最终建议量 |

风险说明：

| 字段 | 说明 |
|------|------|
| 风险原因 | 使用固定模板展示，不接 AI 自动生成 |
| 必断货扣除量 | 本次动作无法挽回的断货损失 |
| 风险类型 | 断货风险、远期库存风险、必断货扣除较大、数据异常 |

不处理后果：

| 字段 | 说明 |
|------|------|
| 预计最早断货日 | 库存投影中首次出现断货损失的日期 |
| 覆盖周期内断货损失 | 覆盖周期内断货损失合计 |
| 断货影响天数 | 覆盖周期内存在断货损失的自然日数量 |
| 预计恢复可售日 | 可选字段，只有后端能稳定计算恢复口径时才展示 |

V0.5 不建议把「预计恢复可售日」作为必填展示项。该字段依赖未来入库节奏、库存投影和恢复口径定义，容易被误读为确定承诺。

库存投影：

| 字段 | 说明 |
|------|------|
| 日期 | D、D+1、D+2 等 |
| 预测销量 | 当日预测销量 |
| 日初库存 | 当日日初可售库存 |
| 可满足销量 | 当日库存可满足的销量 |
| 断货损失 | 当日无法满足的预测销量 |

计算明细：

- 命中规则。
- 命中的安全天数条件。
- 最终安全天数。
- 发货覆盖天数。
- 下单覆盖天数。
- 发货 / 下单预测销量区间。
- 发货 / 下单建议量扣减明细。

## 6. V0.5 建议开发范围

### 6.1 必须做

```text
首页筛选
总览指标
备货建议列表
风险标签
查看明细抽屉
抽屉里的建议动作
抽屉里的风险原因
抽屉里的库存投影
抽屉里的公式拆解
导出 / 生成计划
```

### 6.2 可以做但不强求

```text
不处理后果
风险等级
总览卡片点击过滤
库存投影简图
```

说明：

- 「不处理后果」建议优先展示 `预计最早断货日` 和 `覆盖周期内断货损失`。
- `断货影响天数` 建议作为默认字段，用于替代过度精确的恢复判断。
- `预计恢复可售日` 如后端计算成本较高或业务口径未确认，可不展示。
- 总览卡片点击过滤可以提升体验，但不是试做功能成败的关键。

### 6.3 明确不做

```text
完整处理状态流转
优先处理队列
批量复核
忽略建议
AI 生成风险原因
情景模拟
复杂优先级排序模型
```

## 7. 主要风险与处理建议

### 7.1 风险一：抽屉内容过重

风险说明：

抽屉同时承载建议动作、风险说明、不处理后果、库存投影和计算公式，容易变成一个小页面，导致用户打开后信息压力过大。

处理建议：

```text
固定顶部：SKU 信息 + 风险等级 + 建议动作 + 核心操作
第一段：风险说明 + 不处理后果
第二段：库存投影
第三段：计算明细
```

绘制原型时要保证固定顶部和第一段能直接回答：

```text
现在建议做什么？
为什么要做？
不做有什么影响？
```

交互建议：

- 抽屉顶部固定，不随内容滚动消失。
- 顶部保留锚点导航：`建议动作`、`库存投影`、`计算明细`。
- 核心操作按钮固定在顶部或第一段内，避免用户滚到公式区后找不到下一步动作。
- 公式拆解默认放在第三段，必要时可折叠展示。

### 7.2 风险二：不处理后果可能计算成本偏高

风险说明：

`预计恢复可售日` 可能依赖未来到货、库存投影和动作生效时间，后端计算成本高于简单字段展示。

该字段不是简单取某个 ETA，而是要持续模拟未来每日库存：

```text
当日日初库存 + 当日预计入库 - 当日预测销量
```

并且需要先明确「恢复」口径，例如：

```text
库存重新大于 0
能满足当天预测销量
连续若干天无断货
恢复到安全库存水位
```

如果口径不清晰，展示恢复日期容易让用户误以为这是确定承诺。

处理建议：

V0.5 优先展示：

```text
预计最早断货日
覆盖周期内断货损失
断货影响天数
```

`预计恢复可售日` 仅作为可选字段。后端无法稳定计算或业务未确认恢复口径时，不展示该字段。

### 7.3 风险三：风险原因与建议动作不可追溯

风险说明：

如果风险原因和建议动作使用 AI 自动生成，会导致解释不可控、口径不稳定，验收和追责困难。

处理建议：

V0.5 使用固定模板：

```text
riskCode + params -> 风险原因文案
suggestedActionCode -> 建议动作文案
```

示例：

```text
STOCKOUT_BEFORE_SHIP_ARRIVAL:
D ~ D+6 存在预计断货损失，合计 180 件。

CREATE_SHIPMENT_PLAN:
建议优先生成发货计划。
```

### 7.4 风险四：首页仍然可能像结果表

风险说明：

首页采用 V0 结构，容易退化成一张大表，用户需要自己判断优先处理谁。

处理建议：

V0.5 不做完整优先处理队列，但至少做到：

```text
主表默认按风险优先级排序。
风险标签视觉突出。
建议发货量和建议下单量视觉突出。
异常风险可快速过滤。
```

## 8. 后端输出建议

V0.5 不需要完整工作流，但后端建议直接输出抽屉所需字段。

```json
{
  "sku": "A1",
  "site": "NA",
  "transportMethod": "快船",
  "riskLevel": "HIGH",
  "riskCodes": [
    "STOCKOUT_BEFORE_SHIP_ARRIVAL",
    "HIGH_STOCKOUT_DEDUCTION_RATIO"
  ],
  "suggestedActionCode": "CREATE_SHIPMENT_PLAN",
  "shipSuggestedQty": 320,
  "orderSuggestedQty": 860,
  "earliestStockoutDate": "D",
  "stockoutDateRange": "D ~ D+6",
  "stockoutLossQty": 180,
  "stockoutImpactDays": 6,
  "estimatedRecoverDate": null,
  "shipCoverageDays": 87,
  "orderCoverageDays": 141,
  "shipForecastRange": "D ... D+86",
  "orderForecastRange": "D ... D+140",
  "inventoryProjection": [
    {
      "date": "D",
      "forecastSales": 80,
      "startAvailableInventory": 50,
      "fulfillableSales": 50,
      "stockoutLoss": 30
    }
  ]
}
```

字段说明：

- `stockoutImpactDays` 为 V0.5 建议默认输出字段，表示覆盖周期内存在断货损失的自然日数量。
- `estimatedRecoverDate` 为可选字段；当后端无法稳定计算或业务未确认恢复口径时，返回 `null` 或不返回该字段。

## 9. 风险原因与建议动作模板

### 9.1 风险原因模板

| 风险码 | 文案模板 |
|------|----------|
| `STOCKOUT_BEFORE_SHIP_ARRIVAL` | `{stockoutDateRange} 存在预计断货损失，合计 {stockoutLossQty} 件。` |
| `HIGH_STOCKOUT_DEDUCTION_RATIO` | `必断货扣除量占覆盖周期预测销量比例较高，请关注到货节奏。` |
| `FUTURE_INVENTORY_RISK` | `部分库存预计到达时间较晚，虽然已在建议量中扣减，但短期仍可能断货。` |
| `DATA_MISSING` | `预测、库存或在途数据缺失，建议人工复核。` |
| `SUGGESTED_QTY_ZERO` | `当前库存及在途库存已覆盖目标周期，建议量为 0。` |

### 9.2 建议动作模板

| 动作码 | 展示文案 |
|------|----------|
| `CREATE_SHIPMENT_PLAN` | 建议优先生成发货计划 |
| `CREATE_PURCHASE_PLAN` | 建议生成采购计划 |
| `REVIEW_INBOUND_ETA` | 建议复核在途到达时间 |
| `CHECK_DATA` | 建议检查预测或库存数据 |
| `NO_ACTION` | 暂不处理，继续观察 |

## 10. 验收关注点

V0.5 验收时重点检查：

- 首页能通过筛选查询备货建议。
- 总览指标与列表数据一致。
- 列表中建议发货量、建议下单量、风险标签展示清楚。
- 点击列表行或「查看明细」能打开右侧抽屉。
- 抽屉第一屏能看到建议动作、风险说明和不处理后果。
- 抽屉能展示库存投影和断货损失。
- 抽屉能展示安全天数、覆盖天数和预测销量区间。
- 所有预测销量区间均使用 `D ~ D+N-1` 口径。
- 风险原因和建议动作来自固定模板，不依赖 AI 自动生成。
- 导出和生成计划入口可见。

---

## 11. 结论

V0.5 的产品边界为：

```text
首页保持 V0，负责快速看结果。
抽屉吸收 V1，负责解释、风险和动作建议。
不做完整工作台和状态流转。
```

该方案适合试做阶段推进，开发成本可控，同时比纯 V0 更容易建立用户对建议量的信任。
