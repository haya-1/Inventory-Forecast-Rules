# 备货建议页面原型绘制指南

## 1. 文档目的

本文用于指导「备货建议页面」原型绘制，分为 V0 和 V1 两个版本。

- V0：用于试做验证，重点验证计算结果、公式解释和风险提示是否能被业务理解。
- V1：用于正式工作台方向，重点支持优先级判断、处理状态流转和批量操作闭环。

所有涉及预测销量求和的区间统一采用：

```text
D ~ D+N-1
```

其中：

- D 表示计算日 / 决策日。
- N 表示覆盖天数。
- 覆盖 87 天时，预测销量区间为 `D ... D+86`。
- 覆盖 141 天时，预测销量区间为 `D ... D+140`。

## 2. V0 原型绘制指南

### 2.1 V0 定位

V0 是试做版本，目标不是做完整工作台，而是验证：

```text
我要发多少？
我要下多少？
为什么是这个数？
有没有风险？
```

V0 页面应该让用户快速看到建议量，并能通过明细抽屉理解建议量来源。

### 2.2 V0 页面结构

```text
备货建议页
├─ 筛选区
├─ 总览指标
├─ 备货建议列表
├─ 计算明细抽屉
└─ 库存投影区域
```

### 2.5 V0 绘制要点

- 主表只保留关键字段，不把所有公式字段塞入列表。
- `建议发货量` 和 `建议下单量` 需要视觉突出。
- 风险列用标签展示：`断货风险`、`远期库存风险`、`数据异常`、`正常`。
- 点击列表行或「查看明细」打开右侧抽屉。
- 库存投影可以先做简化图，不必支持复杂交互。
- 主表默认按风险优先级排序，其次按建议发货量或建议下单量降序排序。
- 所有求和区间统一使用 `D ~ D+N-1`。

## 3. V1 原型绘制指南

### 3.1 V1 定位

V1 是正式工作台方向，目标从「展示计算结果」升级为「驱动补货处理」。

V1 页面需要回答：

```text
今天先处理谁？
应该采取什么动作？
处理后状态是什么？
不处理会有什么后果？
```

### 3.2 V1 页面结构

```text
备货建议工作台
├─ 筛选区
├─ 今日处理总览
├─ 优先处理队列
├─ 备货建议列表
├─ 处理状态管理
└─ 右侧决策抽屉
   ├─ 建议动作
   ├─ 不处理后果
   ├─ 计算明细
   ├─ 库存投影
   └─ 操作闭环
```

### 3.6 V1 绘制要点

- 首页不再只是「结果列表」，而是「今日处理台」。
- 总览指标需要可点击，点击后过滤列表。
- 主表默认按风险等级和建议动作排序。
- 每条建议必须有处理状态。
- 右侧抽屉第一屏优先展示「建议动作」和「不处理后果」，公式放在后面。
- 库存投影放进抽屉内，并和单个 SKU 绑定。
- 支持批量处理：批量生成发货计划、批量生成采购计划、批量导出。
- 保留计算透明度，但不要让公式压过决策信息。

## 4. V0 / V1 差异总结

| 项目 | V0 | V1 |
|------|----|----|
| 页面定位 | 备货建议结果页 | 备货建议工作台 |
| 核心目标 | 验证计算、解释结果 | 驱动日常处理 |
| 主表重点 | 建议发货量、建议下单量、风险 | 风险等级、建议动作、处理状态 |
| 明细重点 | 公式拆解、命中规则、库存投影 | 建议动作、不处理后果、库存投影、处理闭环 |
| 操作能力 | 查看、导出、生成计划 | 批量处理、复核、忽略、状态流转 |
| 适用阶段 | 试做、内部验证、业务确认规则 | 正式上线、日常运营使用 |

建议推进节奏：

```text
先画 V0：
确认字段、公式、风险提示、抽屉信息是否被业务接受。

再画 V1：
在 V0 被认可后，加处理队列、状态流转、批量操作和不处理后果。
```
