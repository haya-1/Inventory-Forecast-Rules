# 销量预测计算过程 — 需求说明

## 1. 文档目的与范围

本文档描述**补货详情**「销量预测」Tab 所依据的**销量预测计算逻辑**（业务规则与展示需求），供产品、前端与后端对齐。  
**促销相关销量**在首期逻辑中**暂不纳入**基础日均与系数计算（后续版本可单列）。

**关联功能**：

- 规则配置：「销量预测规则」页面配置的去噪项、日均权重、影响系数等。
- 展示：补货详情页「计算过程解析」——竖向「地铁图」结构，每步需包含**公式说明**、**元数据**、**计算结果数据**。

---

## 2. 总体计算流程（三阶段）

对某一**父体**，在选定**预测规则**且规则**启用**、且规则适用范围包含该父体（或配置为「全部父体」）时，按下述顺序计算：

```
近 90 天原始实际销量
    → 【阶段一】销量去噪 → 得到每日销量集合 A（90 个日粒度值）
    → 【阶段二】日销量规则（权重加权）→ 基础日均 X（不含促销）
    → 【阶段三】影响系数连乘 → 未来各自然日的预测日销量（可形成按日横轴的预测曲线等）
```

### 2.1 符号约定

| 符号 | 含义 |
|------|------|
| A | 去噪后的近 90 天每日销量集合，记为 \(A = \{a_1, a_2, \ldots, a_{90}\}\)（日期与下标一一对应滑动窗口） |
| X | 基础日均（由多窗口日均加权得到） |
| \(w_k\) | 规则配置的「近 k 日」权重，\(k \in \{3,7,14,30,60,90\}\)（具体以产品配置为准），且满足 \(\sum w_k = 100\%\) |
| \(\mu_k\) | 在集合 A 上统计得到的「近 k 日」日均 |
| \(K^{(d)}_t\) / 各维 K | 各维度影响系数在**未来第 t 个自然日**上取到的值（无配置或该日无命中时按 1 或产品默认） |

---

## 3. 阶段一：销量去噪（四类规则 → 集合 A）

### 3.1 输入

- **时间窗口**：近 **90** 个自然日（或业务约定的 D-89～D0）的**原始实际日销量**。
- **规则元数据**：来自当前适配预测规则中的「去噪」配置，共 **四类**（与配置模块一一对应）：

1. **大订单去噪**（`autoDenoise`，配置区亦称「日销量自动去噪」）  
   - 元数据字段（与规则页一致）：**订单销量阈值** `x`、**计入用的固定销量** `y`、`largeEnabled`（是否启用）。  
   - **业务规则（需求描述）**：在**订单粒度**判断——若某笔订单中某产品的销量 **≥ 阈值 `x`**，则该笔订单中该产品贡献的销量在后续汇总为日销量时 **改为固定值 `y`**（不是「近 x 日均 × 倍数」的日销截断）。若同日多单，逐笔判断后再汇总至日粒度。  
   - 若同一天与其它去噪同时命中，覆盖顺序见 3.3。

2. **指定日期去噪 — 固定值**（`dateDenoise` 中类型为固定值）  
   - 元数据：起止日期 \([d_0, d_1]\)、替换值 \(v\)。  
   - **规则**：该区间内每日销量替换为 \(v\)（与近 90 天取交集的天数参与计算）。

3. **指定日期去噪 — 百分比**（`dateDenoise` 中类型为百分比）  
   - 元数据：起止日期、百分比 \(p\%\)。  
   - **规则**：该区间内每日销量 × \(p/100\)。

4. **退货量去噪**（`returnDenoise`）  
   - 元数据：起止日期、退货率 \(pct\%\) 等。  
   - **规则**：在约定区间内按退货相关逻辑对日销做平滑或折扣（具体公式以实现/算法文档为准，需求侧要求**可解释、可追溯**）。

### 3.2 输出

- 集合 **A**：长度与滑动窗口一致的一列**修正后日销量**，作为阶段二输入。

### 3.3 同日多规则命中时的处理顺序（需求约定）

若同一自然日命中多条去噪规则，建议处理顺序为：

**大订单去噪 → 日期类去噪（固定值 / 百分比，按配置顺序或产品约定顺序）→ 退货去噪**

（最终以产品签认顺序为准；前端展示需在「计算过程解析」中写明当次采用的顺序。）

---

## 4. 阶段二：日销量规则 — 加权基础日均 X（不含促销）

### 4.1 公式

在集合 **A** 上，分别计算近 3 / 7 / 14 / 30 / 60 /（ optional 90）日的**日均** \(\mu_3, \mu_7, \ldots\)（仅对窗口内有完整 k 日的情形按产品定义处理边界）。

基础日均：

\[
X = \sum_{k \in K} w_k \cdot \mu_k
\]

其中 \(K\) 为启用权重的窗口集合，且 **\(\sum_k w_k = 100\%\)**（配置校验在规则保存时完成）。

### 4.2 业务说明

- **促销**：首期**不参与**上述加权（若后续纳入，需单独 PRD 说明与 X 的关系）。
- **元数据**：各 \(w_k\) 与规则「日均销量权重」配置一致。  
- **输出**：标量 **X**，并建议同时输出各 **\(w_k \cdot \mu_k\)** 分项供页面展示。

---

## 5. 阶段三：影响系数连乘（按「未来自然日」产出预测日销量）

### 5.1 总公式（需求层）

阶段二得到的是**单一标量**「基础日销量 **X**」。阶段三在此基础上，对未来预测期内的**每一个自然日 \(t\)**，将当日各维度系数连乘，得到该日的**预测日销量**（不含促销与其它业务校正时）：

\[
\hat{a}_t \approx X \times K^{\text{季节}}_t \times K^{\text{同比}}_t \times K^{\text{生命周期}}_t \times K^{\text{竞对}}_t \times K^{\text{目标}}_t \times \cdots
\]

- **\(t\)**：业务约定的**未来预测区间**内的自然日（与补货详情曲线横轴「按日」一致）。  
- **维度集合**以当前规则配置为准；未启用维度或当日不适用时系数视为 **1**（或与算法文档一致的默认）。

### 5.2 时间粒度与区间语义（与「销量预测规则」配置对齐）

1. **系数区间精度**：同比、生命周期、竞对、目标等分段配置的**起止均为日期**（精确到自然日，`yyyy-mm-dd`），不再使用「仅到月」的粒度。

2. **区间为预测期生效**：上述各段配置的 \([d_0, d_1]\) 表示：在**未来预测**时，该系数对**落入该日期区间内的每一个自然日**分别生效；用于解释「预测期每一天」的系数来源，**不与**「近 90 天历史窗口」混作同一套区间语义（历史窗口仅用于阶段一、二）。

3. **季节系数**：仍按**公历月**维护 12 个系数；对任意未来日 \(t\)，\(K^{\text{季节}}_t\) 由 \(t\) 所在月份映射得到（再与当日其它维度连乘）。

4. **同日多段冲突**：若同一自然日 \(t\) 在同一维度命中多条配置，**优先级 / 是否禁止重叠** 由产品与算法文档约定；实现需保证**每日至多一个有效系数**或可解释的合成规则。

5. **展示**：补货详情「计算过程解析」可对某**代表日**给出合并后的连乘示例；完整曲线则按日输出 \(\hat{a}_t\)。

### 5.3 与配置模块的对应（示例）

| 维度 | 配置来源（示例） |
|------|------------------|
| 季节 | 12 个月季节系数（按日映射到当月） |
| 同比 | 多段 **起止日期** + 系数 |
| 生命周期 | 次新品/老品 + **起止日期** + 系数 |
| 竞对 | **起止日期** + 系数 |
| 目标 | **起止日期** + 系数 |

## 6. 与「补货详情」页面的展示需求

### 6.1 入口与规则绑定

- 父体从补货列表进入补货详情（`code` 查询参数）。  
- **适用规则**：在启用规则中，筛选「适用范围包含本父体」或「全部父体」的规则；多规则时展示策略由产品定义（列表展示 vs 单选主规则等）。

### 6.2 「计算过程解析」展示结构（地铁图）

- **版式**：竖向时间线 / **地铁图**，自上而下对应阶段一 → 二 → 三。  
- **每一站**须包含：  
  - **公式解释**（可读数学/文字描述）；  
  - **元数据**（来自当前规则快照或接口返回）；  
  - **本步计算结果**（数值、集合统计、分项表等）。  

### 6.3 前端演示范围（当前实现约定）

- 产品上可约定：**仅对部分父体/规则组合**提供完整示例数据（如父体 `LFT00002261` + 规则 `SF20260410001`），其它组合展示占位或「后续接入」提示。

## 7. 非功能与待确认项

| 项 | 说明 |
|----|------|
| 精度 | 中间量与最终对外字段的小数位、舍入规则由算法/后端统一约定 |
| 与图表关系 | 「销量预测结果」曲线横轴为**自然日**，纵轴为各日**预测日销量**，应与阶段三按日输出 \(\hat{a}_t\) 一致；演示页可用示意数据，上线后需与本文链路一致 |
| 促销 | 是否进入 X、是否独立乘子，待独立需求 |
| 多时区/自然日 | 自然日以业务主数据口径为准 |
