销量预测计算过程 — 需求说明

1. 文档目的与范围

本文档描述补货详情「销量预测」Tab 所依据的销量预测计算逻辑（业务规则与展示需求），供产品、前端与后端对齐。
促销相关销量在首期逻辑中暂不纳入基础日均与系数计算（后续版本可单列）。

关联功能：

  • 规则配置：「销量预测规则」页面配置的去噪项、日均权重、影响系数等。
  • 展示：补货详情页「计算过程解析」——竖向「地铁图」结构，每步需包含公式说明、元数据、计算结果数据。

2. 总体计算流程（三阶段）

对某一父体，在选定预测规则且规则启用、且规则适用范围包含该父体（或配置为「全部父体」）时，按下述顺序计算：

近 90 天原始实际销量
    → 【阶段一】销量去噪 → 得到每日销量集合 A（90 个日粒度值）
    → 【阶段二】日销量规则（权重加权）→ 基础日均 X（不含促销）
    → 【阶段三】影响系数连乘 → 预测日均（及可外推的预测曲线等）

A 去噪后的近 90 天每日销量集合，记为 A={a1,a2,…,a90}A = \{a_1, a_2, \ldots, a_{90}\}A={a1​,a2​,…,a90​}（日期与下标一一对应滑动窗口）

3. 阶段一：销量去噪（四类规则 → 集合 A）

3.1 输入

  • 时间窗口：近 90 个自然日（或业务约定的 D-89～D0）的原始实际日销量。
  • 规则元数据：来自当前适配预测规则中的「去噪」配置，共 四类（与配置模块一一对应）：

  1. 大订单去噪（autoDenoise，配置区亦称「日销量自动去噪」）

    • 元数据字段（与规则页一致）：订单销量阈值 x、计入用的固定销量 y、largeEnabled（是否启用）。
    • 业务规则（需求描述）：在订单粒度判断——若某笔订单中某产品的销量 ≥ 阈值 x，则该笔订单中该产品贡献的销量在后续汇总为日销量时 改为固定值 y（不是「近 x 日均 × 倍数」的日销截断）。若同日多单，逐笔判断后再汇总至日粒度。
    • 若同一天与其它去噪同时命中，覆盖顺序见 3.3。

  2. 指定日期去噪 — 固定值（dateDenoise 中类型为固定值）

    • 元数据：起止日期 [d0,d1][d_0, d_1][d0​,d1​]、替换值 vvv。
    • 规则：该区间内每日销量替换为 vvv（与近 90 天取交集的天数参与计算）。

  3. 指定日期去噪 — 百分比（dateDenoise 中类型为百分比）

    • 元数据：起止日期、百分比 p%p\%p%。
    • 规则：该区间内每日销量 × p/100p/100p/100。

  4. 退货量去噪（returnDenoise）

    • 元数据：起止日期、退货率 pct%pct\%pct% 等。
    • 规则：在约定区间内按退货相关逻辑对日销做平滑或折扣（具体公式以实现/算法文档为准，需求侧要求可解释、可追溯）。

3.2 输出

  • 集合 A：长度与滑动窗口一致的一列修正后日销量，作为阶段二输入。

3.3 同日多规则命中时的处理顺序（需求约定）

若同一自然日命中多条去噪规则，建议处理顺序为：

大订单去噪 → 日期类去噪（固定值 / 百分比，按配置顺序或产品约定顺序）→ 退货去噪

（最终以产品签认顺序为准；前端展示需在「计算过程解析」中写明当次采用的顺序。）

4. 阶段二：日销量规则 — 加权基础日均 X（不含促销）

4.1 公式

在集合 A 上，分别计算近 3 / 7 / 14 / 30 / 60 /（ optional 90）日的日均 μ3,μ7,…\mu_3, \mu_7, \ldotsμ3​,μ7​,…（仅对窗口内有完整 k 日的情形按产品定义处理边界）。

基础日均：

X=∑k∈Kwk⋅μkX = \sum_{k \in K} w_k \cdot \mu_kX=k∈K∑​wk​⋅μk​

其中 KKK 为启用权重的窗口集合，且 ∑kwk=100%\sum_k w_k = 100\%∑k​wk​=100%（配置校验在规则保存时完成）。

4.2 业务说明

  • 促销：首期不参与上述加权（若后续纳入，需单独 PRD 说明与 X 的关系）。
  • 元数据：各 wkw_kwk​ 与规则「日均销量权重」配置一致。
  • 输出：标量 X，并建议同时输出各 wk⋅μkw_k \cdot \mu_kwk​⋅μk​ 分项供页面展示。

5. 阶段三：影响系数连乘

5.1 总公式（需求层）

预测日均（示意，不含促销与其它业务校正时）：

预测日均≈X×K季节×K同比×K生命周期×K竞对×K目标×⋯\text{预测日均} \approx X \times K_{季节} \times K_{同比} \times K_{生命周期} \times K_{竞对} \times K_{目标} \times \cdots预测日均≈X×K季节​×K同比​×K生命周期​×K竞对​×K目标​×⋯

（维度集合以当前规则配置为准，未列维度系数视为 1 或不适用。）

5.2 系数取值规则（关键点）

  1. 无时间段覆盖：在近统计窗口内，若某维度没有任何一条规则的时间段与窗口重合，则该维度 K = 1。

  2. 单段完全覆盖：若一段配置完全覆盖统计窗口内相关子区间，则该维度 K 取配置系数（或按产品定义的聚合方式）。

  3. 多段覆盖或部分覆盖：若同一维度在近窗口内存在多段不同系数，或单段不能完全覆盖近 90 天：

    • 需先按自然日拆解窗口；
    • 对每个自然日取当日适用系数（冲突时按产品优先级）；
    • 再按天数加权平均得到该维度的有效系数 K；
    • 最后参与连乘。

  4. 各维度 K 均为标量：每个维度输出一个参与连乘的 K，界面需展示「元数据摘要 + 有效 K 的计算说明」。

5.3 与配置模块的对应（示例）

维度 配置来源（示例）
季节 12 个月季节系数

6. 与「补货详情」页面的展示需求

6.1 入口与规则绑定

  • 父体从补货列表进入补货详情（code 查询参数）。
  • 适用规则：在启用规则中，筛选「适用范围包含本父体」或「全部父体」的规则；多规则时展示策略由产品定义（列表展示 vs 单选主规则等）。

6.2 「计算过程解析」展示结构（地铁图）

  • 版式：竖向时间线 / 地铁图，自上而下对应阶段一 → 二 → 三。
  • 每一站须包含：
    • 公式解释（可读数学/文字描述）；
    • 元数据（来自当前规则快照或接口返回）；
    • 本步计算结果（数值、集合统计、分项表等）。

6.3 前端演示范围（当前实现约定）

  • 产品上可约定：仅对部分父体/规则组合提供完整示例数据（如父体 LFT00002261 + 规则 SF20260410001），其它组合展示占位或「后续接入」提示。
