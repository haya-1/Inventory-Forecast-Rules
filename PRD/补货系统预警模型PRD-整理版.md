## 4. 下单预警模型

### 4.1 核心变量定义

根据原文，下单预警模型涉及以下核心变量：

- 当前总可用供应量  
  `当前总可用供应量 = FBA库存（在库 + 预留 + 在途）+ CN仓 + 采购在途 + 预发货 + 海外仓库存`

- 下单覆盖总时长  
  `下单覆盖总时长 = 合同审批 + 生产周期 + 品检入库时间 + 提货准备 + 运输时间 + 安全库存天数`

### 4.2 时间序列触发逻辑

根据原文，系统从今天（Day 0）开始，逐日累加未来的计划年规销量，直到累加总量等于或刚好超过当前总可用供应量。此时所经历的天数，即为动态可售天数。

### 4.3 触发条件

当满足以下条件时，触发下单预警：

`动态可售天数 < 下单覆盖总时长`

## 5. 补货预警模型

### 5.1 动态库存计算定义

根据原文，补货预警模型采用时间序列方式进行库存推演。

- `S(t)`：第 `t` 天的预测销量（根据年规，按照月平摊到每天）
- `Inv(t)`：第 `t` 天的预测库存
- 第 `t` 天的库存总量：  
  `Inv(t) = Inv(t-1) - S(t) + 到货量(t)`

### 5.2 预警触发：断货点探测

#### 5.2.1 逻辑描述

系统从当前日期开始，逐日推算未来的库存走势，找到第一个 `Inv(t) ≤ 0` 的日期，该日期即为断货点 `T0`。

#### 5.2.2 计算方式

原文表述为：寻找最小的 `t`，使得 `Inv(t) ≤ 0`。

### 5.3 预警触发条件

根据原文，断货点 `T0` 与不同运输时效的关系决定预警类型：

1. 如果 `快船总运输时间 ≤ T0 ≤ 普船总运输时间`，则触发`快船救火预警`。
2. 如果 `普船总运输时间 ≤ T0 ≤ 普船总运输时间 + 发货天数 + 安全库存天数`，则触发`普船补货预警`。
3. 如果 `T0 ≤ 快船总运输时间`，则触发`必断货`，按`快船 + 普船`方式同步走。
