# 预警模块 · 钉钉机器人通知设计规格

> 版本：v3.0（ActionCard 版）
> 更新日期：2026-06-21
> 配套原型：`预警钉钉通知设计.html`

## 1. 设计概述

### 1.1 通知渠道与消息类型

所有预警通知通过**钉钉自定义机器人**推送，消息类型为 **ActionCard**（独立跳转按钮模式）。采用单条独立推送方式（不做汇总聚合），每条通知对应一条预警规则的触发事件。

### 1.2 ActionCard 能力与约束

| 能力 | 支持情况 | 本设计的应对 |
|------|----------|-------------|
| 彩色文字 | 不支持 | 用加粗标识异常值 |
| 顶部色带 | 不支持 | — |
| 表格排版 | 不支持 | 用 Markdown 无序列表「**标签：** 值」逐行排列 |
| 分割线 | 不支持 | 用空行分段 |
| 加粗 | 支持 `**text**` | 用于强调异常值 |
| 链接 | 支持 `[text](url)` | 暂不使用，按钮已覆盖跳转 |
| 独立跳转按钮 | 支持 1~5 个 | 每种预警配 1 个「查看详情」按钮 |

### 1.3 卡片通用结构

ActionCard 消息由以下部分组成：

1. **title**（通知摘要）—— 显示在钉钉消息列表和锁屏通知中，格式为「{预警模型名} | {一句话异常描述}」
2. **text**（Markdown 正文）—— 显示在卡片展开后，包含：
   - 三级标题（`###`）+ 完整标题
   - 无序列表形式的字段键值对
3. **btns**（按钮数组）—— 单个「查看详情」按钮，灰色背景

### 1.4 标题设计原则

每种预警类型使用个性化标题模板，格式统一为「预警模型名 | 一句话异常描述」，将核心异常指标直接嵌入标题，接收人在消息列表即可判断紧急程度。标题建议不超过 30 字。

| 预警模型 | 标题模板 | 示例 |
|----------|----------|------|
| 本地库龄预警 | 本地库龄预警 \| {SKU}库龄≥{N}天超阈值 | 本地库龄预警 \| ST18106DENIMBLUE095库龄≥90天超阈值 |
| FBA库龄预警 | FBA库龄预警 \| {ASIN}库龄≥{N}天超阈值 | FBA库龄预警 \| B083J69DSQ库龄≥365天超阈值 |
| FBA长期仓储费预警 | FBA长期仓储费预警 \| {ASIN}距收费日{D}天 | FBA长期仓储费预警 \| B07NV7QK7H距收费日12天 |
| 销售偏差预警 | 销售偏差预警 \| {SPU}预测偏差{±X}% | 销售偏差预警 \| WT82527258预测偏差+22.5% |
| 供应商生产延期预警 | 供应商生产延期预警 \| {SPU}交期延后{D}天 | 供应商生产延期预警 \| ST223501交期延后12天 |
| 达成偏差预警 | 达成偏差预警 \| {SPU}达成率{R}% | 达成偏差预警 \| HKA19达成率82.3% |
| SKC销售偏差预警 | SKC销售偏差预警 \| {SKC}占比差{Z}% | SKC销售偏差预警 \| HK1839-BLK占比差22.5% |

## 3. ActionCard JSON 通用报文结构

所有预警通知共用以下 JSON 结构，`text` 和 `btns` 字段按上述模板填充：

```json
{
  "msgtype": "actionCard",
  "actionCard": {
    "title": "{预警模型名} | {一句话异常描述}",
    "text": "### {完整标题}\n\n- **字段1：** 值1\n- **字段2：** 值2\n...",
    "btnOrientation": "1",
    "btns": [
      {
        "title": "查看详情",
        "actionURL": "https://oms.example.com/alert/detail?id={预警ID}"
      }
    ]
  }
}
```

**关键字段说明：**

| 字段 | 说明 |
|------|------|
| `title` | 通知摘要，显示在钉钉消息列表和锁屏通知中。建议不超过 30 字。 |
| `text` | Markdown 格式正文。支持 `###` 标题、`**加粗**`、`- 列表`、`[链接](url)`。不支持表格、彩色文字、分割线。 |
| `btnOrientation` | `"0"` = 按钮竖直排列，`"1"` = 按钮水平排列。本设计统一使用 `"1"`。 |
| `btns[].title` | 按钮文案，统一为「查看详情」 |
| `btns[].actionURL` | 按钮跳转地址。建议用 `dingtalk://dingtalkclient/page/link?url={编码后URL}` 格式以在钉钉内打开。 |

## 4. 通知接收人规则

通知接收人在预警规则配置页中设定，支持两种方式：

1. **指定人员** —— 从人员列表中选择（可多选），包括个人和群组
2. **链接负责人** —— 系统根据父体所属链接的运营负责人自动确定

两者二选一。通知方式固定为钉钉。

## 5. 推送频率与去重

- 每条预警规则触发时独立推送一条通知
- 同一规则对同一父体/SKU，在 24 小时内不重复推送（可配置冷却时间）
- 不同规则触发的通知互不干扰

## 6. 按钮交互说明

- **查看详情**：跳转到 OMS 系统内的预警详情页面，展示该条预警的完整信息和处理记录
- **按钮样式**：灰色背景（`#f0f0f0`），与钉钉卡片默认按钮风格一致
- **跳转方式**：建议使用 `dingtalk://dingtalkclient/page/link?url=` 协议，在钉钉客户端内打开页面，避免跳出到浏览器
