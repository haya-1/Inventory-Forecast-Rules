# 备货建议页面父体维度改版需求

## 1. 背景

备货建议页面由子体维度列表调整为父体维度列表。主页面用于查看每个父体的发货建议量和下单建议量，详情抽屉用于查看该父体下各子体的建议量与计算明细。

本页面不承接建议动作、异常处理闭环和预警处理闭环。远期库存风险、数据异常风险、处理优先级、系统建议动作等内容后续由预警模块或数据校验模块补充。

发货建议详情内允许展示 SKU 维度发货窗口，该字段只用于解释当前发货建议的处理优先度，不作为主页面风险字段，也不触发系统建议动作。

## 2. 页面定位

页面回答：

```text
每个父体建议发多少？
每个父体建议下多少？
这些建议量由哪些子体建议量合计而来？
建议量的计算依据是什么？
发货建议中每个 SKU 的发货窗口是什么？
哪些父体因为数据或配置不完整没有生成建议？
```

页面不回答：

```text
应该优先处理哪个父体或风险？
不处理会有什么后果？
远期库存风险、数据异常风险如何处理？
系统建议用户采取什么动作？
```

## 3. 核心口径

### 3.1 展示粒度

- 主页面列表粒度：父体 + 计算来源。
- 主页面通过“发货建议 / 下单建议”tab 分开展示两个业务动作。
- 年规目标和预测日销量两种计算来源在同一个列表中展示，不再提供数据来源切换。
- 详情抽屉不展示 tab 组件，发货建议和下单建议分段展示。
- 详情抽屉展示标题汇总、发货建议明细、下单建议明细、计算明细。
- 详情抽屉不再单独展示“汇总信息”正文分段和“物流方式拆分”正文分段。

### 3.2 计算粒度

备货建议量仍以子体为最小计算单元。

```text
父体发货建议总量 = Σ 子体发货最终建议量
父体下单建议总量 = Σ 子体下单最终建议量
```

父体维度不重新计算预测销量曲线，不重新套用库存扣减公式。

### 3.3 数据完整性

配置数据不完整、关键数据缺失、无法命中规则、物流时效缺失等情况，不生成备货建议记录。

```text
不进入主列表
不参与父体合计
不展示为风险或异常
进入“未生成建议”入口
```

## 4. 主页面需求

### 4.1 筛选区

| 字段 | 说明 |
|------|------|
| 父体 / 子体 | 支持输入父体编号或子体 / SKU 查询 |
| 运营负责人 | 按父体运营负责人过滤 |
| 适用规则 | 按命中的备货规则过滤 |
| 只看有建议量 | 仅展示当前 tab 下建议量大于 0 的父体 + 计算来源记录 |

不展示：

- 决策日期 D。
- 物流方式筛选。
- 站点筛选。

### 4.2 计算批次信息

筛选区下方展示当前建议量的计算上下文：

| 字段 | 说明 |
|------|------|
| 计算时间 | 当前建议结果的计算完成时间 |
| 数据来源 | 当前建议结果包含的计算来源，展示为年规目标 + 预测日销量 |
| 规则版本 | 当前建议结果使用的备货规则编号 |
| 未生成建议 | 未进入主列表的父体数量，点击查看明细 |

### 4.3 汇总指标

| 指标 | 说明 |
|------|------|
| 建议发货总量 | 当前筛选条件下父体发货建议量合计 |
| 建议下单总量 | 当前筛选条件下父体下单建议量合计 |

不展示父体数量、子体数量等数量统计。

### 4.4 建议类型切换

“备货建议列表”标题下方展示 tab 按钮：

| Tab | 列表口径 | 批量操作 |
|-----|----------|----------|
| 发货建议 | 展示父体发货备货周期与发货建议量 | 批量生成发货 |
| 下单建议 | 展示父体下单备货周期与下单建议量 | 批量生成采购 |

切换 tab 后：

- 主列表只展示当前 tab 对应的备货周期和建议量。
- “只看有建议量”按当前 tab 的建议量判断。
- 主列表排序按当前 tab 的建议量降序。
- 勾选与批量生成只作用于当前 tab 的可见记录。

| 备货周期 | 当前 tab 对应的覆盖周期 / 安全天数 |

## 5. 未生成建议入口

### 5.1 入口

主页面计算批次信息右侧展示“未生成建议：N”。

点击后打开弹窗，展示未进入主列表的父体。

### 5.2 字段

| 字段 | 说明 |
|------|------|
| 父体编号 | 未生成建议的父体 |
| 站点 | 父体所属站点 |
| 运营负责人 | 父体运营负责人 |
| 未生成原因 | 未生成建议的直接原因 |
| 处理方向 | 用户后续应前往规则配置、数据校验或计算任务中处理 |

### 5.3 原因枚举

- 备货规则维护不完整。
- 规则未启用。
- 缺少年规/预测日销量。
- 缺少FBA/本地库存数据。

未生成建议不作为风险展示。

## 6. 详情抽屉需求

### 6.1 抽屉结构

详情抽屉采用固定顶部、内容滚动、底部固定操作栏。

顶部展示：

- 父体编号。
- 匹配规则。
- 计算来源。
- 运营负责人。
- 计算时间。
- 父ASIN / SPU。
- 站点。
- 发货口径。
- 下单口径。
- 物流方式。
- 发货建议总量。
- 下单建议总量。

内容分段：

- 发货建议明细。
- 下单建议明细。
- 计算明细。

底部操作：

- 生成发货计划。
- 生成采购计划。
- 导出明细。

### 6.2 发货建议明细

发货建议明细用于解释当前父体发货建议量由哪些子体、物流方式和到货时间构成。

展示要求：

- 发货建议和下单建议分开展示，暂不使用详情 tab 组件。
- 发货建议构成中，建议量为 0 的 SKU 不展示。
- 发货单据按物流方式分组展示，每个物流方式默认收起。
- 发货单据不展示矩阵图。
- 发货单据组标题展示“物流方式 · 共 N 个 SKU · 合计 X 双 · 到货时间 YYYY-MM-DD”。
- 详情底部固定展示生成计划类按钮。

发货建议构成字段：

| 字段 | 说明 |
|------|------|
| 子体 / SKU | 当前子体 |
| ASIN | 子体对应 ASIN |
| 命中规则 | 当前子体命中的安全库存条件；超出部分支持展开查看 |
| 物流方式 | 子体建议量对应物流方式 |
| 到货时间 | 当前物流方式按决策日 D + 物流时效计算的预计到货日期 |
| 安全天数 | 子体最终安全天数 |
| SKU 发货窗口 | 当前子体发货窗口，仅发货建议展示 |
| 发货建议量 | 子体发货最终建议量 |

发货单据字段：

| 字段 | 说明 |
|------|------|
| SKU | 当前子体 SKU |
| seller-SKU | 卖家 SKU；原型中可与 SKU 保持一致 |
| 条码 / FNSKU | 发货识别码；原型中可使用 X 开头的模拟条码 |
| 到货时间 | 当前物流方式预计到货日期 |
| 发货数量 | 当前 SKU 在该物流方式下的发货数量 |

### 6.3 下单建议明细

下单建议明细用于解释当前父体下单建议量由哪些子体构成。

采购建议构成字段：

| 字段 | 说明 |
|------|------|
| 子体 / SKU | 当前子体 |
| ASIN | 子体对应 ASIN |
| 命中规则 | 当前子体命中的安全库存条件；超出部分支持展开查看 |
| 安全天数 | 子体最终安全天数 |
| 下单建议量 | 子体下单最终建议量 |

采购单据字段：

| 字段 | 说明 |
|------|------|
| SKU | 当前子体 SKU |
| seller-SKU | 卖家 SKU；原型中可与 SKU 保持一致 |
| 条码 / FNSKU | 采购识别码；原型中可使用 X 开头的模拟条码 |
| 采购数量 | 当前 SKU 下单数量 |

采购单据不展示 ASIN、命中规则、安全天数。

### 6.4 发货窗口

发货窗口只在详情抽屉的发货建议构成中展示，主页面不展示发货窗口字段，采购建议不展示发货窗口字段。

发货窗口以 SKU + 计算来源为粒度，根据当前库存预计断货点与物流可补救时间窗判断。

| 等级 | 名称 | 口语副文案 | 判定口径 | 展示含义 |
|------|------|------------|----------|----------|
| A | A：窗口已过 | 已赶不上 | 断货点早于最快物流到货时间 | 发货窗口已经错过，现有库存无法撑到最快物流到货，立即发货用于缩短断货期 |
| B1 | B1：末班窗口 | 仅剩末班 | 断货点处于最快物流到货时间至最快物流到货时间 + 发货频率之间 | 本期发货是最后窗口，错过本轮发货即转为 A |
| B2 | B2：窗口收紧 | 快线可赶 | 断货点处于最快物流到货时间 + 发货频率至最慢物流到货时间之间 | 慢速物流已赶不上，应优先安排较快物流 |
| C | C：窗口充足 | 全都可赶 | 断货点晚于最慢物流到货时间，或覆盖期内不断货 | 最慢物流也能赶上，可按常规节奏安排 |

等级命名采用「发货时间窗口」角度，口语副文案用于悬浮提示等辅助展示，不参与计算。

表头“SKU 发货窗口”旁展示说明图标，悬浮后展示以上等级含义。

### 6.5 计算明细字段

父体层展示合计口径：

```text
父体发货建议总量 = Σ 子体发货最终建议量
父体下单建议总量 = Σ 子体下单最终建议量
```

计算明细按步骤折叠 / 展开展示。默认可收起，每一步标题左侧展示序号图标，右侧展示该步骤摘要值，例如“13.25 件/天”“156 天”“库存抵扣 1,830 件”。

子体层展示必要计算摘要：

| 字段 | 说明 |
|------|------|
| 子体 / SKU | 当前子体 |
| 预测销量 | 覆盖周期内预测销量合计 |
| 库存扣减 | 覆盖周期内可用于抵扣的库存合计 |
| 扣减项 | 展示参与建议量计算的库存类扣减项；发货建议不展示必断货扣除量，下单建议也不展示必断货扣除量 |
| 最终建议 | 子体最终建议量 |

“库存扣减”表头旁展示说明图标，悬浮后说明包含库存：FBA 可用、覆盖周期内预计可用的 FBA 在途、本地仓库存、覆盖周期内预计可用的已下单未入库等。

发货建议和采购建议的库存抵扣表均保留完整库存列，包括平台仓、物流在途、本地仓库存、采购在途、总库存。发货建议中的本地仓库存列展示为：

```text
本地仓库存(预发货+剩余) = 总量(预发货库存 + 剩余库存)
```

采购建议中的本地仓库存列展示普通本地仓库存数值，不拆分预发货库存和剩余库存。该展示不改变发货建议量与采购建议量各自的库存扣减口径。

多物流方式分配表展示物流方式、计算优先级、物流时效、到货时间、负责区间、缺口销量、建议量。

覆盖周期拆解表展示负责区间、区间天数、区间缺口销量、归属物流方式，不展示到货时间。

预计结果以图表方式展示库存变化，不在库存走势图下方额外展示解释说明。

## 7. 数据流

```text
子体级备货建议计算结果
→ 过滤掉配置或数据不完整对象
→ 按筛选条件过滤子体结果
→ 按父体 + 计算来源分别聚合发货建议量和下单建议量
→ 根据当前 tab 渲染发货建议列表或下单建议列表、汇总指标、计算批次信息
→ 点击父体查看子体明细和计算明细
```

聚合规则：

- 发货建议量：对子体发货最终建议量求和。
- 下单建议量：对子体下单最终建议量求和。
- 发货备货周期：取父体下子体发货覆盖周期中的最大覆盖周期，并展示对应安全天数。
- 下单备货周期：取父体下子体下单覆盖周期中的最大覆盖周期，并展示对应安全天数。
- 当前筛选条件变化后，先过滤子体，再重新聚合父体。
- 年规目标和预测日销量分别形成独立记录，不在父体行内合并。
- 当前 tab 为发货建议时，主列表展示发货备货周期和发货建议量。
- 当前 tab 为下单建议时，主列表展示下单备货周期和下单建议量。
- 主列表不聚合、不展示 SKU 发货窗口；发货窗口只在详情发货建议构成中按 SKU 展示。
- 子体级计算结果需携带物流方式、计算优先级、到货时间、负责区间、缺口销量、建议量，用于详情计算明细展示。

## 8. 操作入口

页面保留以下用户主动操作：

- 发货建议 tab 下，主列表勾选父体 + 计算来源记录后批量生成发货计划。
- 下单建议 tab 下，主列表勾选父体 + 计算来源记录后批量生成采购计划。
- 主列表查看详情。
- 详情抽屉生成发货计划。
- 详情抽屉生成采购计划。
- 详情抽屉导出明细。
- 主页面导出当前结果。

以上操作不作为系统建议动作展示，仅作为用户可执行命令。

## 9. 验收标准

- 主页面列表一行代表一个父体 + 计算来源。
- 主页面通过“发货建议 / 下单建议”tab 分开展示。
- 发货建议 tab 只展示发货备货周期和发货建议量。
- 下单建议 tab 只展示下单备货周期和下单建议量。
- 主页面展示计算时间、数据来源、规则版本。
- 主页面不提供年规目标 / 预测日销量切换控件。
- 主页面在匹配规则后展示计算来源列。
- 年规目标和预测日销量两种计算结果在同一个 tab 列表中同时展示。
- 主页面提供未生成建议入口，并能查看未生成原因。
- 主列表不展示单行生成按钮，生成计划需先勾选父体后批量生成。
- 主页面不展示父体数量、子体数量、子体数。
- 主页面不展示物流方式筛选。
- 主列表不展示物流方式列。
- 主页面不展示风险相关字段、建议动作字段、库存与销量投影。
- 父体 + 计算来源的发货建议量等于详情中同计算来源下所有子体发货建议量之和。
- 父体 + 计算来源的下单建议量等于详情中同计算来源下所有子体下单建议量之和。
- 详情抽屉不展示 tab 组件，发货建议和下单建议分段展示。
- 发货建议构成不展示建议量为 0 的 SKU。
- 发货建议构成展示 SKU 发货窗口列，等级为 A：窗口已过、B1：末班窗口、B2：窗口收紧、C：窗口充足。
- SKU 发货窗口表头提供悬浮说明，说明每个等级的判定含义。
- 发货单据按物流方式分组，默认不展开，不展示矩阵图。
- 发货单据字段包含 SKU、seller-SKU、条码 / FNSKU、到货时间、发货数量。
- 采购单据字段包含 SKU、seller-SKU、条码 / FNSKU、采购数量，不展示 ASIN、命中规则、安全天数。
- 计算明细步骤支持折叠 / 展开，标题右侧展示该步骤摘要值。
- 多物流方式分配表展示到货时间，覆盖周期拆解表不展示到货时间。
- 库存扣减表头提供悬浮说明，说明具体包含哪些库存。
- 详情抽屉底部固定展示生成发货计划、生成采购计划、导出明细。
- 配置或数据不完整的对象不生成备货建议记录，不进入主列表。

## 10. 后续模块边界

以下系统级能力不在备货建议页面内实现，后续由预警模块、数据校验模块或任务日志承接：

- 主页面风险排序与处理优先级。
- 断货风险预警提醒。
- 远期库存风险识别与提醒。
- 数据异常识别与提醒。
- 不处理后果分析。
- 预警优先级。
- 风险处理闭环。
- 未生成建议的批量修复闭环。
