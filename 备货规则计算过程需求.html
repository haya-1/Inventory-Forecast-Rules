#备货规则计算过程 — 需求说明

#1. 文档目的与范围

本文档描述「备货规则」的计算口径与结果展示要求，供产品、前端与后端对齐。

本文以《备货规则配置页面 — PRD》为基准文件。若两份文档口径冲突，以《备货规则配置页面 — PRD》为准。

备货规则不再使用「备货天数 × 单一日销量」的静态口径，而是基于销量预测模块输出的从计算日开始的每日预测销量曲线，按覆盖周期对预测日销量求和，得到发货建议量与下单建议量。

本文覆盖：

  • 预测销量求和区间。
  • 安全天数、采购时效、物流时效的计算关系。
  • 发货建议量与下单建议量的覆盖周期。
  • 多运输方式下的建议量优先级分配计算。
  • 库存扣减、缺口销量分摊、库存投影断货风险、发货风险评级。
  • 计算结果输出字段。

本文不覆盖：

  • 海外仓 / 中转模式。当前版本彻底隐藏，不参与展示、配置与计算。
  • 箱规、整箱、MOQ 等采购约束。当前建议量仅向上取整数。
  • 站点、仓库、运输方式与父体 / SKU 的最终绑定逻辑。该部分后续接入。

#2. 预测销量输入

备货规则消费销量预测模块输出的每日预测销量：

\[\hat{a}_{D}, \hat{a}_{D+1}, \ldots, \hat{a}_{D+n-1}\]

其中：

符号 含义
\(D\) 计算日 / 决策日
\(\hat{a}_{D+i}\) 从计算日起第 \(i+1\) 个自然日的预测日销量，\(i\) 从 0 开始
\(N\) 当前备货场景的覆盖天数

覆盖 N 天时，预测销量求和为：

\[F(N) = \sum_{i=0}^{N-1} \hat{a}_{D+i}\]

积分区间固定为：

D ~ D+N-1

示例：

覆盖天数 = 70 天
预测销量求和 = Σ forecastSales(D ... D+69)

注意：

  • 覆盖天数从计算日 D 当天开始计算。
  • 不再使用旧口径 D+1 ~ D+N。

#3. 规则配置项与派生参数

#3.1 采购时效

采购时效模块包含四个配置项：

配置项 默认值 说明
采购频率 15 天 下一次采购机会间隔
采购交期 30 天 从下单到供应端交付的周期
提货准备 3 天 到货后可发货前的准备时间
质检 2 天 到货后质检处理时间

计算时拆分为两个口径：

采购交付提前期 = 采购交期 + 提货准备 + 质检

采购周期覆盖天数 = 采购交付提前期 + 采购频率

含义：

  • 采购交付提前期用于判断本次下单动作最早何时形成 CN 仓 / 本地仓可发库存。
  • 采购频率只纳入下单覆盖周期，不纳入本次下单动作的最早可用时间。

示例：

采购交付提前期 = 30 + 3 + 2 = 35 天
采购周期覆盖天数 = 35 + 15 = 50 天

#3.2 安全库存天数

FBA 默认安全天数默认值：

28 天

当父体 / SKU 命中多个安全库存条件时：

最终安全天数 = max(默认安全天数, 所有已启用命中条件的安全天数)

示例：

命中条件 安全天数
默认安全天数 28 天
B - 旺款 28 天
新品（0-3个月） 45 天
AD 广告流量 ASIN 28 天

最终安全天数：

max(28, 28, 45, 28) = 45 天

#3.3 直发物流时效

当前版本仅支持直发模式：

CN 仓 / 本地仓 → FBA 仓

物流配置项：

  • 站点。
  • 本地仓发货频率。
  • 运输方式。
  • 计算优先级。
  • CN 仓至 FBA 物流时效。

默认配置示例：

站点 发货频率 运输方式 计算优先级 物流时效
北美站 7 天 美森正班 1 37 天
北美站 7 天 普船 2 57 天
欧洲站 7 天 普船 1 45 天
欧洲站 7 天 空运 2 15 天

同一站点可配置多种运输方式。计算时需先按计算优先级从高到低排序；数字越小优先级越高。若优先级相同，按物流时效从短到长作为兜底排序。

#4. 总体计算流程

对某个父体 / SKU，在命中有效销量预测规则与备货规则后，按如下流程计算：

销量预测曲线
    → 确定最终安全天数
    → 读取站点物流方式与发货频率
    → 计算发货覆盖天数 / 下单覆盖天数
    → 对 D ~ D+N-1 的预测日销量求和
    → 计算库存投影与发货风险评级
    → 发货扣减 FBA 库存 / FBA 在途；下单扣减库存、在途和已下单未入库
    → 计算发货建议总量 / 下单建议总量
    → 若存在多运输方式，按计算优先级依次认领可覆盖日期并拆分建议量
    → 向上取整并输出结果
    → 输出异常与风险提示

#5. 发货建议量

#5.1 业务含义

发货建议量用于回答：

当前应从 CN 仓 / 本地仓发多少货到 FBA？

#5.2 覆盖天数

发货动作最早可用天数：

发货最早可用天数 = CN仓至FBA时效

发货覆盖天数：

发货覆盖天数 =
CN仓至FBA时效 + 安全天数 + CN仓发货频率

当同一站点只配置一种运输方式时，按该运输方式物流时效计算。

当同一站点配置多种运输方式时，发货覆盖周期以最长物流时效作为周期终点基础：

发货覆盖天数 =
最长CN仓至FBA时效 + 安全天数 + CN仓发货频率

#5.3 单运输方式计算公式

发货覆盖周期预测销量 =
Σ forecastSales(D ... D+发货覆盖天数-1)

发货原始建议量 =
发货覆盖周期预测销量
− FBA库存
− FBA在途

发货最终建议量 =
max(0, ceil(发货原始建议量))

#5.4 多运输方式优先级分配计算

当同一站点配置多种运输方式时，系统需要按运输方式分别输出发货建议量。

分配规则：

第 k 种运输方式发货最早可用天数 = 该运输方式 CN仓至FBA时效

第 k 种运输方式理论可覆盖区间 =
D + 第 k 种运输方式发货最早可用天数 ~ D + 发货覆盖天数 - 1

第 k 种运输方式实际负责区间 =
理论可覆盖区间内尚未被更高优先级运输方式覆盖的日期

示例：

空运 15 天，普船 45 天
发货覆盖天数 = 91 天，即覆盖 D ~ D+90
优先级：空运 = 1，普船 = 2

空运建议量覆盖：D+15 ~ D+90
普船建议量覆盖：0（可覆盖日期已被更高优先级空运覆盖）

若优先级调整为普船 = 1，空运 = 2：

普船建议量覆盖：D+45 ~ D+90
空运建议量覆盖：D+15 ~ D+44

说明：

  • 空运到货日前，即 D ~ D+14 的预测销量不归属任何运输方式建议量，只用于库存投影和发货风险评级。
  • 运输方式到货前累计销量需要参与库存投影，影响该运输方式到货时的库存缺口，但不直接计入该运输方式建议量。
  • 当覆盖周期内所有可被运输方式覆盖的日期均已被认领后，停止后续低优先级运输方式计算。

多运输方式下：

发货建议总量 = Σ 各运输方式发货建议量

库存扣减口径：

  • FBA 库存、FBA 在途不能在每种运输方式上重复扣减。
  • 系统应基于库存投影或统一缺口分摊口径，将可用库存优先用于较早日期需求，再按运输方式优先级认领日期段得到各运输方式建议量。
  • 各运输方式建议量求和应等于发货最终建议量；若因向上取整产生极小差异，差异落到最后一种运输方式。

#6. 下单建议量

#6.1 业务含义

下单建议量用于回答：

当前应向供应端 / 采购端下单多少？

#6.2 覆盖天数

下单动作最早可用天数：

下单最早可用天数 =
采购交付提前期 + CN仓至FBA时效

下单覆盖天数：

下单覆盖天数 =
采购周期覆盖天数 + CN仓至FBA时效 + 安全天数 + CN仓发货频率

当同一站点只配置一种运输方式时，按该运输方式物流时效计算。

当同一站点配置多种运输方式时，下单覆盖周期以最长物流时效作为周期终点基础：

下单覆盖天数 =
采购周期覆盖天数 + 最长CN仓至FBA时效 + 安全天数 + CN仓发货频率

#6.3 单运输方式计算公式

下单覆盖周期预测销量 =
Σ forecastSales(D ... D+下单覆盖天数-1)

下单原始建议量 =
下单覆盖周期预测销量
− FBA库存
− FBA在途
− CN仓可用
− 已下单未入库

下单最终建议量 =
max(0, ceil(下单原始建议量))

#6.4 多运输方式优先级分配计算

当同一站点配置多种运输方式时，系统需要按运输方式分别输出下单建议量。

分配规则：

第 k 种运输方式下单最早可用天数 =
采购交付提前期 + 第 k 种运输方式 CN仓至FBA时效

第 k 种运输方式理论可覆盖区间 =
D + 第 k 种运输方式下单最早可用天数 ~ D + 下单覆盖天数 - 1

第 k 种运输方式实际负责区间 =
理论可覆盖区间内尚未被更高优先级运输方式覆盖的日期

示例：

采购交付提前期 = 35 天
空运 15 天，普船 45 天
下单覆盖天数 = 130 天，即覆盖 D ~ D+129
优先级：空运 = 1，普船 = 2

空运下单最早可用日：D+50
普船下单最早可用日：D+80

空运建议量覆盖：D+50 ~ D+129
普船建议量覆盖：0（可覆盖日期已被更高优先级空运覆盖）

若优先级调整为普船 = 1，空运 = 2：

普船建议量覆盖：D+80 ~ D+129
空运建议量覆盖：D+50 ~ D+79

说明：

  • 最短下单最早可用日前，即示例中的 D ~ D+53，不归属任何运输方式建议量，仅用于库存投影和风险提示，不作为扣减项影响下单建议量。
  • 运输方式到货前累计销量需要参与库存投影，影响该运输方式到货时的库存缺口，但不直接计入该运输方式建议量。
  • 当覆盖周期内所有可被运输方式覆盖的日期均已被认领后，停止后续低优先级运输方式计算。

多运输方式下：

下单建议总量 = Σ 各运输方式下单建议量

库存扣减口径：

  • FBA 库存、FBA 在途、CN 仓可用、已下单未入库不能在每种运输方式上重复扣减。
  • 系统应基于库存投影或统一缺口分摊口径，将可用库存优先用于较早日期需求，再按运输方式优先级认领日期段得到各运输方式建议量。
  • 各运输方式建议量求和应等于下单最终建议量；若因向上取整产生极小差异，差异落到最后一种运输方式。

#7. 库存扣减口径

#7.1 发货建议量扣减项

发货建议量扣减：

  • FBA 库存。
  • FBA 在途。

FBA 在途建议按预计到货时间参与库存投影，仅将覆盖周期内可用的在途库存纳入本次建议量扣减。

#7.2 下单建议量扣减项

下单建议量扣减：

  • FBA 库存。
  • FBA 在途。
  • CN 仓可用。
  • 已下单未入库。

FBA 在途与已下单未入库建议按预计到货时间参与库存投影，仅将覆盖周期内可用的库存纳入本次建议量扣减。

#7.3 缺口销量计算口径

多物流方式拆分时，系统分配的不是原始预测销量，而是库存投影后形成的缺口销量。

每日库存投影口径：

某日可用库存 = 前一日结余库存 + 当日可用入库

某日可满足销量 = min(某日可用库存, 当日预测销量)

某日缺口销量 = max(0, 当日预测销量 - 某日可用库存)

某日结余库存 = max(0, 某日可用库存 - 当日预测销量)

区间缺口销量：

区间缺口销量 = Σ 区间内每日缺口销量

多物流方式下：

  • 库存和在途不能在每个物流方式上重复扣减。
  • 可用库存优先覆盖更早日期需求。
  • 物流方式按计算优先级认领其可覆盖日期段内的缺口销量。
  • 运输方式到货前的累计销量只参与库存投影和发货风险评级，不直接归属为该物流方式建议量，也不作为扣减项影响建议量。

示例：

日期区间 预测销量 库存抵扣 缺口销量
D ~ D+14 90 90 0
D+15 ~ D+44 180 120 60
D+45 ~ D+90 120 0 120

若普船优先级高于空运：

物流方式 到货时间 实际负责区间 分配缺口
普船 D+45 D+45 ~ D+90 120
空运 D+15 D+15 ~ D+44 60

若空运优先级高于普船：

物流方式 到货时间 实际负责区间 分配缺口
空运 D+15 D+15 ~ D+90 180
普船 D+45 无剩余区间 0

#7.4 远期库存风险边界

若计算层按预计到货时间过滤库存，仅扣减覆盖周期内可用的在途库存，则覆盖周期外远期库存不会压低本次建议量，可不单独输出远期库存风险。

若后续仍采用全量扣减 FBA 在途、已下单未入库等库存口径，短期断货但建议量为 0 的风险由预警模块或库存投影模块承接，不作为备货建议主流程的必需输出。

#8. 库存投影与断货风险边界

#8.1 推导方式

系统可基于库存投影推断预计断货点与断货损失，用于解释库存走势、发货风险评级或后续预警模块。

发货建议量和下单建议量均不再扣减“必断货扣除量”。也就是说，断货损失只作为风险信息，不作为建议量计算扣减项。

库存投影口径：

当日日初可售库存 =
max(0, 前一日期末可售库存) + 当日预计可用入库

当日可满足销量 =
min(当日日初可售库存, 当日预测销量)

当日断货损失 =
当日预测销量 − 当日可满足销量

当日期末可售库存 =
max(0, 当日日初可售库存 − 当日预测销量)

等价地，当日断货损失也可写为：

当日断货损失 =
min(当日预测销量, max(0, 当日预测销量 − 当日日初可售库存))

默认口径：

  • 当日预计可用入库在当日销售前可用。
  • 当某日当日断货损失大于 0，可记录为预计断货日或断货风险日。
  • 若数据源口径为日末入库，库存投影展示需按实际数据源口径调整。

#8.2 计算边界

发货必断货扣除量 = 不计算
下单必断货扣除量 = 不计算

发货建议量与下单建议量只扣减各自库存类扣减项：

  • 发货建议量扣减 FBA 库存、FBA 在途。
  • 下单建议量扣减 FBA 库存、FBA 在途、CN 仓可用、已下单未入库。

#8.3 展示要求

若库存投影出现断货风险，页面可展示：

  • 预计断货日期范围。
  • 断货日期内每日预测销量。
  • 断货日期内每日可满足销量。
  • 断货日期内每日断货损失。

提示文案：

库存投影存在预计断货风险，请关注库存走势与发货风险评级；该风险不改变本次建议量。

#9. 异常提示规则

#9.1 建议量小于等于 0

若原始建议量小于或等于 0：

最终建议量展示为 0

提示文案：

当前库存及覆盖周期内可用在途库存已覆盖目标周期，建议量为 0，请关注库存积压风险。

#9.2 断货风险提示边界

断货风险不再作为建议量扣减项，也不在备货建议主流程内输出“必断货扣除量较大”类提示。

若库存投影显示短期断货，但建议量因库存覆盖、在途到货或其他扣减项计算为 0，该风险由发货风险评级、库存投影说明或后续预警模块承接。

#9.3 发货风险评级

发货风险评级用于在备货建议详情中解释 SKU 发货紧急程度，仅用于发货建议，不用于下单建议，不改变发货建议量、下单建议量和多物流拆分结果。

评级粒度：

SKU + 计算来源

基础变量：

变量 含义
fastestLeadDays 当前命中站点下参与计算物流方式的最短物流时效
slowestLeadDays 当前命中站点下参与计算物流方式的最长物流时效
shipFrequencyDays 当前命中站点的发货频率
rescueLeadDays fastestLeadDays + shipFrequencyDays
stockoutDay 基于库存投影得到的 SKU 预计断货点

库存投影口径：

风险评级库存 = FBA 可用 + 覆盖周期内预计可用的 FBA 在途

系统按预测日销量或年规目标日销量消耗风险评级库存，得到 stockoutDay。若覆盖周期内未断货，则按最低风险处理。

评级规则：

等级 名称 判断条件 含义
A A：必断货 stockoutDay < fastestLeadDays 最快物流到货前已经断货，本次发货无法完全避免断货
B B：紧急补货 fastestLeadDays <= stockoutDay < rescueLeadDays 最快物流到货后不久即断货，需要尽快安排发货
C C：优先补货 rescueLeadDays <= stockoutDay < slowestLeadDays 断货点早于最慢物流到货，应优先安排较快物流或尽快发货
D D：常规可达 stockoutDay >= slowestLeadDays 或覆盖周期内未断货 常规发货节奏可覆盖当前风险

若当前站点仅配置一种物流方式，则 fastestLeadDays = slowestLeadDays，C 级区间为空，系统按 A / B / D 判断。

#10. 计算过程展示

补货详情页建议展示「备货计算过程解析」。每个步骤需支持折叠 / 展开，标题左侧展示步骤序号，标题右侧展示该步骤的核心结果值。

建议包含：

  1. 预测销量曲线来源。
  2. 安全天数命中明细与最终取值。
  3. 发货覆盖天数拆解。
  4. 下单覆盖天数拆解。
  5. 预测销量求和区间：
D ~ D+N-1
  1. 多运输方式下按计算优先级认领的覆盖区间。
  2. 库存扣减明细，表头统一展示为「库存扣减」，并通过提示说明包含 FBA 库存、FBA 在途、CN 仓可用、已下单未入库等库存项。
  3. 库存投影断货点推导。
  4. 原始建议量、向上取整后的建议量。
  5. 发货风险评级。
  6. 异常提示。

库存抵扣步骤标题右侧需展示库存扣减合计值，例如：

库存抵扣    1,830 件

多物流方式分配表中需展示每种物流方式的到货时间；覆盖周期拆解表不展示到货时间，避免两个表的信息重复。

#12. 验收标准

#12.1 计算验收

  • 预测销量求和区间为：
D ~ D+N-1
  • 采购交付提前期按以下公式计算：
采购交期 + 提货准备 + 质检
  • 采购周期覆盖天数按以下公式计算：
采购交付提前期 + 采购频率
  • 发货最早可用天数按以下公式计算：
CN仓至FBA时效
  • 下单最早可用天数按以下公式计算：
采购交付提前期 + CN仓至FBA时效
  • 单运输方式发货覆盖天数按以下公式计算：
CN仓至FBA时效 + 安全天数 + CN仓发货频率
  • 单运输方式下单覆盖天数按以下公式计算：
采购周期覆盖天数 + CN仓至FBA时效 + 安全天数 + CN仓发货频率
  • 多运输方式发货覆盖周期以最长 CN 仓至 FBA 时效作为周期终点基础。
  • 多运输方式下单覆盖周期以采购周期覆盖天数 + 最长 CN 仓至 FBA 时效作为周期终点基础。
  • 同一站点配置多种运输方式时，发货建议量需要按运输方式计算优先级依次认领可覆盖日期并输出拆分明细。
  • 同一站点配置多种运输方式时，下单建议量需要按运输方式计算优先级依次认领可覆盖日期并输出拆分明细。
  • 多运输方式下，若高优先级运输方式已覆盖全部可覆盖日期，后续低优先级运输方式建议量为 0。
  • 多运输方式下，运输方式到货前累计销量只参与库存投影和发货风险提示，不直接计入该运输方式建议量，也不作为扣减项影响建议量。
  • 多运输方式建议量的库存扣减项不得在每种运输方式上重复扣减。
  • 多运输方式建议量按库存投影后的缺口销量分配，不按原始预测销量重复分配。
  • 多运输方式拆分明细需输出物流方式、优先级、物流时效、到货时间、负责区间和建议量。
  • 建议量向上取整数。
  • 最终建议量不得小于 0。
  • 安全天数取默认安全天数与所有命中条件安全天数中的最高值。
  • 当日断货损失基于当日预测销量、当日日初可售库存、当日可满足销量推导。
  • 发货建议量与下单建议量均不扣减必断货扣除量。
  • 断货损失仅用于库存投影、发货风险评级或后续预警模块，不改变建议量。

#12.2 提示验收

  • 原始建议量小于等于 0 时，页面展示 0 并提示库存覆盖风险。
  • 发货建议需按 SKU 输出 A / B / C / D 风险评级，且该评级不改变建议量。
  • 库存投影断货风险不作为建议量扣减项，后续可由预警模块或库存投影模块承接。
  • 远期库存风险不作为备货建议主流程必需提示项，后续由预警模块或库存投影模块承接。
